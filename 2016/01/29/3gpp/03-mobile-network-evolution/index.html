    1. 移动通信代际划分概述

    移动通信的代际（Generation）划分以 ITU（国际电信联盟）制定的 IMT 系列标准为官方依据，各代的核心诉求从”能通话”逐步演进为”高速数据”再到”万物互联”。

    代际 代表标准 商用时间 核心能力 核心技术
    1G AMPS / NMT / TACS 1980s 模拟语音 FDMA
    2G GSM / IS-95 1991 数字语音 + SMS TDMA / CDMA
    2.5G GPRS 1999 低速分组数据（~115 kbps） GPRS over GSM
    2.75G EDGE 2003 中速数据（~384 kbps） 8-PSK
    3G UMTS/WCDMA 2001 移动宽带（~2 Mbps） DS-CDMA
    3.5G HSDPA / HSUPA 2005 增强移动宽带（~14 Mbps） AMC + HARQ
    3.75G HSPA+ 2008 高速移动宽带（~42 Mbps） MIMO + 64QAM
    4G LTE（Release 8） 2009 真正移动宽带（~150 Mbps） OFDMA + MIMO
    4.5G LTE-Advanced（Release 10） 2013 IMT-Advanced（~1 Gbps） 载波聚合
    4.9G LTE-Advanced Pro（Release 13+） 2016 进一步增强 NB-IoT / eMTC

    注意：3GPP 本身不使用”3.5G/4.9G”等非官方命名，这些叫法是业界惯用简称，官方依据是 3GPP Release 编号。

    2. 2G：GSM 与数字蜂窝的奠基

    2.1 GSM 的诞生背景

    1G 时代，各国采用不兼容的模拟标准（如美国 AMPS、北欧 NMT、英国 TACS），导致无法跨国漫游，频谱效率低下，且易被窃听。欧洲各国在 1982 年由 CEPT（欧洲邮电行政大会）成立了 Groupe Spécial Mobile（GSM）工作组，目标是制定一套统一的泛欧数字蜂窝标准。1989 年，标准化职责移交至 ETSI，并将 GSM 重新诠释为 Global System for Mobile Communications（全球移动通信系统）。

    1991 年，芬兰运营商 Radiolinja 率先商用 GSM 网络，随后 GSM 迅速在全球普及，成为有史以来采用最广泛的移动通信标准。

    2.2 GSM 网络架构

    GSM 网络在逻辑上分为三个子系统：

    BSS（基站子系统） 负责无线接入侧：

    • BTS（Base Transceiver Station）：基站收发信机，负责与 MS 的无线通信，一个 BTS 覆盖一个小区（Cell）。
    • BSC（Base Station Controller）：基站控制器，管理多个 BTS，处理无线资源分配、切换（Handover）、功率控制等。BTS 与 BSC 之间通过 Abis 接口连接。

    NSS（Network Switching Subsystem）/ 核心网 负责交换与数据库：

    • MSC（Mobile Switching Center）：移动交换中心，负责电话呼叫的路由与交换，是 NSS 的核心节点。
    • GMSC（Gateway MSC）：网关 MSC，作为 PLMN 与 PSTN/ISDN 之间的接口，所有入呼均通过 GMSC 查询 HLR 后转发。
    • HLR（Home Location Register）：归属位置寄存器，存储用户的永久签约信息（IMSI、MSISDN、服务权限、当前 VLR 地址等）。
    • VLR（Visitor Location Register）：访问位置寄存器，与 MSC 紧密耦合，存储当前驻留在该 MSC 区域内的用户临时信息（TMSI、位置区等）。
    • AuC（Authentication Center）：鉴权中心，存储 SIM 根密钥 Ki，生成鉴权三元组（RAND, SRES, Kc）用于认证和加密。
    • EIR（Equipment Identity Register）：设备标识寄存器，管理 IMEI 白名单/黑名单，防止非法设备接入。

    主要接口：

    接口名 连接节点 协议
    Um MS ↔ BTS GSM 空口协议
    Abis BTS ↔ BSC LAPD over E1/T1
    A BSC ↔ MSC BSSAP（SS7）
    B MSC ↔ VLR MAP
    C GMSC ↔ HLR MAP
    D HLR ↔ VLR MAP

    2.3 GSM 关键技术

    多址接入：FDMA + TDMA

    GSM 采用频分多址（FDMA）与时分多址（TDMA）相结合的方式：

    • 频域：每个载波信道带宽为 200 kHz，相邻信道间隔 200 kHz。
    • 时域：每个 200 kHz 载波划分为 8 个时隙（Time Slot），形成一个 TDMA 帧（周期 4.615 ms）。
    • 每个用户占用一个时隙，即同一载波最多支持 8 路通话。

    双工方式：FDD

    上行（UL）与下行（DL）使用不同频段，典型频段如 GSM-900（上行 890–915 MHz，下行 935–960 MHz），收发频率间隔 45 MHz。

    信道编码

    语音编码使用 RPE-LTP（Regular Pulse Excitation - Long Term Prediction） 编解码器，速率 13 kbps（后期引入 AMR 自适应多速率编码）。信道编码采用卷积码，并结合交织（Interleaving）提升抗突发误码能力。

    安全机制

    • 认证：网络侧向 UE 发送随机数 RAND，SIM 用 Ki 和 A3 算法生成 SRES 返回，网络侧对比验证。
    • 加密：鉴权通过后，双方以 RAND 和 Ki 通过 A8 算法生成会话密钥 Kc，使用 A5 算法（A5/1 或 A5/2）对无线信道加密。
    • TMSI：通过临时移动用户标识替代 IMSI 在空口传输，保护用户隐私。

    移动性管理

    • 位置区（Location Area, LA）：由一组小区组成，UE 在 LA 内移动无需更新位置（只在 LA 边界触发位置更新 / Location Update）。
    • 寻呼（Paging）：网络在整个 LA 范围内广播寻呼消息通知 UE 有来电。
    • 切换（Handover）：GSM 采用 Mobile Assisted Handover（MAHO），UE 主动测量邻区信号强度并上报，BSC 决策发起切换。

    2.4 2.5G：GPRS 与分组交换的引入

    GSM 原生仅支持电路交换（Circuit Switched），数据业务通过 CSD（Circuit Switched Data）传输，速率仅 9.6 kbps，且占用整个通话时隙。为支持互联网接入，3GPP 在 Release 97/98 引入了 GPRS（General Packet Radio Service）。

    GPRS 的核心思想是在 GSM 网络上叠加一个 分组交换（Packet Switched）域，新增两个核心网元：

    • SGSN（Serving GPRS Support Node）：服务 GPRS 支持节点，负责 PS 域的移动性管理、会话管理、鉴权及路由，是 PS 域的核心控制节点，对应 CS 域的 MSC/VLR。
    • GGSN（Gateway GPRS Support Node）：网关 GPRS 支持节点，是 PLMN 与外部 PDN（如互联网）之间的接口，负责 IP 地址分配与路由，类似今天的 PGW。

    GPRS 允许多个用户共享同一时隙的无线资源（最多可使用 8 个时隙同时传输），理论峰值速率约 171.2 kbps，实际典型速率 56~115 kbps。GPRS 还引入了 PDP Context（Packet Data Protocol Context） 机制，即用户与外部网络之间的虚拟连接，每个 PDP Context 分配一个 IP 地址。

    GPRS 新增的接口：

    接口 连接节点 说明
    Gb BSS ↔ SGSN 帧中继承载，传输分组数据
    Gn SGSN ↔ GGSN IP/GTP 隧道（同一 PLMN）
    Gi GGSN ↔ 外部 PDN 互联网接入点
    Gs SGSN ↔ MSC/VLR CS/PS 域协调

    2.5 2.75G：EDGE 的增强

    EDGE（Enhanced Data rates for GSM Evolution） 在 3GPP Release 99 中定义（核心规范参考 TS 45.001 等），通过更高阶的调制方式进一步提升 GSM 数据速率：

    • 将调制方式从 GMSK（Gaussian Minimum Shift Keying）升级为 8-PSK（8 Phase Shift Keying），每个符号携带 3 bit（GMSK 每符号仅 1 bit），理论峰值速率达 384 kbps。
    • 引入 9 种调制编码方案（MCS-1 ~ MCS-9），支持链路自适应。
    • EDGE 无需更换基站天线，对运营商来说升级成本较低，因此在 3G 部署前被广泛用于过渡。

    3. 3G：UMTS/WCDMA 与宽带移动互联网

    3.1 3G 标准化背景与 IMT-2000

    1992 年，ITU 启动了 IMT-2000（International Mobile Telecommunications-2000） 标准化工作，目标是制定一套统一的 3G 全球标准。ITU 的核心需求包括：

    • 室内环境支持 2 Mbps 数据速率
    • 步行环境支持 384 kbps
    • 车载高速移动支持 144 kbps

    然而，不同地区利益集团推出了多个竞争提案，最终形成五个 IMT-2000 标准：

    标准 标准化组织 主要地区 空口技术
    UMTS/WCDMA 3GPP 欧洲、日本、中国部分 W-CDMA FDD
    CDMA2000 3GPP2 北美、韩国 MC-CDMA
    TD-SCDMA CWTS（→3GPP） 中国（中国移动） CDMA TDD
    DECT ETSI 欧洲（主要室内） —
    WiMAX（后期） IEEE 少数地区 OFDM

    3GPP 在 1998 年 12 月 正式成立，接管并发展 GSM 演进体系，其第一个 3G 规范集称为 Release 99（R99），于 1999 年底冻结，2001 年日本 NTT DoCoMo 率先商用 UMTS。

    3.2 UMTS 网络架构（Release 99）

    UMTS Release 99 的整体架构在 3GPP TS 23.002 中定义，由三个主要部分构成：

    UTRAN（UMTS Terrestrial Radio Access Network） 由若干 RNS（Radio Network Subsystem） 组成，每个 RNS 包括：

    • Node-B：对应 GSM 的 BTS，负责 WCDMA 物理层处理（扩频、解扩、功率测量等），一个 Node-B 服务一个或多个小区。Node-B 与 RNC 之间通过 Iub 接口（与 GSM 的 Abis 接口相比，Iub 接口在 3GPP 中完整定义，支持多厂商互操作）连接。
    • RNC（Radio Network Controller）：无线网络控制器，对应 GSM 的 BSC，负责无线资源管理（RRM）、切换决策、功率控制、UE 上下文管理等。RNC 之间通过 Iur 接口连接，支持软切换时的数据分发。

    核心网 在 Release 99 中基本复用了 GSM/GPRS 的核心网架构，分为 CS 域和 PS 域：

    • CS 域：MSC、GMSC、HLR（新增 UMTS 鉴权功能），承载语音和传统电路业务。
    • PS 域：SGSN、GGSN，承载数据业务，接口从 Gb（帧中继）升级为 Iu-PS（ATM/IP）。

    主要接口汇总：

    接口 连接节点 协议
    Uu UE ↔ Node-B WCDMA 空口
    Iub Node-B ↔ RNC NBAP / ATM
    Iur RNC ↔ RNC RNSAP / ATM
    Iu-CS RNC ↔ MSC RANAP / ATM
    Iu-PS RNC ↔ SGSN RANAP / ATM

    3.3 WCDMA 核心技术

    多址接入：DS-CDMA（直接序列码分多址）

    WCDMA（Wideband CDMA）是 UMTS 的空口技术（规范见 3GPP TS 25.211-25.215 系列）。与 GSM 的 TDMA 不同，CDMA 允许所有用户在 相同时间、相同频率 上同时传输，通过不同的伪随机扩频码（Spreading Code）区分用户：

    • 信道带宽：5 MHz（相比 GSM 的 200 kHz 宽 25 倍）
    • 码片速率（Chip Rate）：3.84 Mcps（百万码片/秒）
    • 扩频增益（Processing Gain）：通过扩频将用户信号带宽扩展，抵抗干扰
    • 双工方式：主要为 FDD（频分双工），也支持 TDD 模式（TD-SCDMA 即为 TDD 变体）

    功率控制（Power Control）

    CDMA 系统中，由于所有用户共用同一频段，某一用户功率过大会对其他用户造成干扰（”远近效应”）。WCDMA 采用两层功率控制：

    • 开环功率控制（Open-Loop）：UE 根据下行信号强度估算上行发射功率初始值
    • 闭环功率控制（Closed-Loop / Inner-Loop）：网络侧每 0.667 ms 发送一次 TPC（Transmit Power Control）命令，以 1 dB 步进调整 UE 发射功率，速率达 1500 Hz（远快于 GSM 的切换速率），是 WCDMA 容量保障的关键机制。
    • 外环功率控制（Outer-Loop）：RNC 根据 BLER（Block Error Rate）目标调整 SIR（Signal-to-Interference Ratio）门限。

    软切换（Soft Handover）

    CDMA 最独特的特性之一。由于所有小区使用相同频率，UE 可以同时与多个 Node-B 保持连接（宏分集），RNC 通过 Iur 接口 将数据发给多个 Node-B，在接收侧进行 MRC（最大比合并），相比 GSM 的”硬切换”（先断后连）具有更低的切换掉话率和更平滑的移动体验。

    调制与速率

    • 下行：QPSK 调制（Release 99 基本速率）
    • 上行：BPSK（以保持低 PAPR）
    • 最大理论速率（Release 99）：2.048 Mbps（室内静止环境，需要完整码道）

    信道结构

    WCDMA 定义了三类信道：

    • 逻辑信道（Logical Channel）：描述传输内容（如 DCCH/DTCH/CCCH 等）
    • 传输信道（Transport Channel）：描述传输方式（如 DCH/FACH/PCH/RACH 等）
    • 物理信道（Physical Channel）：实际空口承载（如 DPCH/CPICH/SCH/AICH 等）

    3.4 3.5G：HSDPA / HSUPA（HSPA）

    HSDPA（High Speed Downlink Packet Access） 在 3GPP Release 5（2002 年）引入，HSUPA（High Speed Uplink Packet Access） 在 Release 6（2004 年）引入，合称 HSPA。

    HSDPA 技术创新

    HSDPA 的根本思想是将调度与重传功能从 RNC 下移至 Node-B，大幅降低时延并提升效率：

    新增物理信道：

    • HS-DSCH（High Speed Downlink Shared Channel）：高速共享下行传输信道，由小区内所有 HSDPA 用户共享，最多使用 15 个码道。
    • HS-SCCH（High Speed Shared Control Channel）：携带调度信息（UE 标识、调制方式、资源分配）。
    • HS-DPCCH（High Speed Dedicated Physical Control Channel）：UE 上报 ACK/NACK 和 CQI（信道质量指示）。

    三大核心技术：

    技术 说明
    AMC（Adaptive Modulation and Coding） 根据 CQI 动态选择调制（QPSK / 16QAM）和码率，信道好时用 16QAM + 高码率，差时退回 QPSK + 低码率
    Node-B 侧快速调度 调度周期 2 ms（TTI = 2 ms），相比 Release 99 的 10/20/40/80 ms TTI 大幅缩短，及时响应信道变化
    HARQ（Hybrid ARQ） 在 Node-B 侧完成重传，软合并（Chase Combining 或 Incremental Redundancy）错误数据包，无需经过 RNC

    HSDPA 峰值速率演进：

    • Release 5：理论最高 14.4 Mbps（16QAM + 15 码道）
    • 实际部署典型值：1.8 / 3.6 / 7.2 Mbps（取决于 UE 类别）

    HSUPA 技术创新（Release 6）

    HSUPA 引入 E-DCH（Enhanced Dedicated Channel），同样将调度功能下移至 Node-B：

    • TTI 缩短为 2 ms 或 10 ms（可配置）
    • 引入 Node-B 控制的上行调度与 HARQ
    • 上行峰值速率达 5.76 Mbps

    3.5 3.75G：HSPA+

    HSPA+（Evolved HSPA） 在 Release 7（2007 年）和 Release 8（2008 年）中引入，进一步增强 HSPA 性能：

    • Release 7 增强：
      • 引入 64QAM 下行调制，峰值速率提升至 21.1 Mbps
      • 引入 2×2 MIMO 支持（下行），最高 28 Mbps
      • 连续分组连接（Continuous Packet Connectivity, CPC）降低信令开销
    • Release 8 增强：
      • 双载波 HSDPA（DC-HSDPA），聚合两个 5 MHz 载波，峰值下行 42 Mbps
      • MIMO + 64QAM 结合，最高 42 Mbps

    HSPA+ 使得运营商能在不大规模部署新基站的情况下提供接近 LTE 初期的用户体验，成为 3G 到 4G 的重要过渡桥梁。

    3.6 IMS 的引入（Release 5）

    Release 5 还引入了 IMS（IP Multimedia Subsystem），这是 3GPP 为实现全 IP 语音/视频业务而设计的统一多媒体框架（规范见 3GPP TS 23.228）。IMS 基于 SIP（Session Initiation Protocol）协议，核心节点包括：

    • P-CSCF（Proxy-CSCF）：UE 的 SIP 信令代理入口点
    • I-CSCF（Interrogating-CSCF）：网络拓扑隐藏，负责选择 S-CSCF
    • S-CSCF（Serving-CSCF）：核心 SIP 服务器，执行注册、会话路由与业务触发
    • HSS（Home Subscriber Server）：3G 中替代 HLR，统一管理用户签约数据

    IMS 的设计超前于其时代，在 LTE 的 VoLTE 和 5G 的 VoNR 中才真正大规模落地，但其架构理念贯穿 4G/5G 始终。

    4. 4G：LTE 与全 IP 时代

    4.1 LTE 的诞生背景与 3GPP Release 8

    2004 年，3GPP 启动了 UMTS 的”长期演进（Long Term Evolution，LTE）”研究项目，核心目标是应对无线业务爆炸式增长（彼时 YouTube、iPhone 即将改变互联网格局），并克服 WCDMA 在频谱效率和时延上的固有局限。

    ITU 的 IMT-Advanced 要求（对应 4G 官方标准）需满足：

    • 低移动性（步行）：1 Gbps 峰值速率
    • 高移动性（车载）：100 Mbps 峰值速率
    • 频谱效率：下行 15 bps/Hz，上行 6.75 bps/Hz

    LTE 的第一个版本 3GPP Release 8 于 2008 年 12 月冻结（标准文本见 TS 36 系列），2009 年 12 月瑞典 TeliaSonera 率先在斯德哥尔摩和奥斯陆商用。LTE 本身严格来说属于”准 4G”，满足 IMT-Advanced 的是后续的 LTE-Advanced（Release 10）。

    4.2 LTE 系统架构（EPS）

    LTE 引入了全新的系统架构：EPS（Evolved Packet System），由 E-UTRAN（接入网）和 EPC（Evolved Packet Core，演进分组核心网） 组成（规范见 3GPP TS 23.401）。

    EPS 最大的架构革新是彻底放弃了 CS 域，所有业务（包括语音）均通过 全 IP 分组交换承载。同时，控制面与用户面明确分离（Control / User Plane Separation，CUPS）。

    核心设计原则：

    • 扁平化接入网：取消 RNC，eNB 直接承担无线资源管理功能，eNB 之间通过 X2 接口 直接通信。
    • 控制面/用户面分离：MME 纯粹处理控制面信令，SGW/PGW 处理用户面数据，实现独立扩展。
    • 全分组交换：无 CS 域，Voice over LTE（VoLTE）通过 IMS 承载。
    • 低时延设计：端到端时延目标 < 5 ms（用户面），空口 TTI 降至 1 ms。

    4.3 E-UTRAN：eNodeB 的扁平化

    eNB（evolved Node-B） 是 LTE 接入网的唯一节点，整合了 UMTS 中 Node-B + RNC 的所有功能：

    eNB 主要职责（规范见 3GPP TS 36.300）：

    • 无线资源管理（RRM）：无线承载控制、接纳控制、连接移动性控制
    • 用户面数据压缩与加密（PDCP 层）
    • 调度：上行/下行数据包调度
    • 移动性信息的测量与上报配置
    • 系统信息广播（SIB）
    • 寻呼消息传输

    X2 接口（eNB ↔ eNB）：支持基于 X2 的切换（不经过 EPC），减少切换时延；用户面数据转发防止切换时的丢包。

    S1 接口（eNB ↔ EPC）：分为 S1-MME（控制面，使用 S1AP/SCTP/IP）和 S1-U（用户面，使用 GTP-U/UDP/IP）。

    4.4 EPC 核心网元详解

    MME（Mobility Management Entity）

    MME 是 EPC 的控制面核心节点（规范见 3GPP TS 23.401 §5.3.3），不处理任何用户面数据，主要负责：

    • NAS 信令终结：与 UE 的 NAS（Non-Access Stratum）层信令（包括 EMM/ESM 消息）在 MME 终结。
    • 鉴权与安全：协调 HSS 执行 EPS AKA（Authentication and Key Agreement）鉴权，管理 NAS/AS 安全密钥。
    • 移动性管理：
      • 追踪区（Tracking Area, TA）管理与 TAU（Tracking Area Update）处理
      • 空闲态（ECM-IDLE）UE 的位置管理
      • 切换过程中的 MME 选择与重定向
    • 会话管理：协调 SGW/PGW 完成 EPS Bearer 的建立、修改、释放。
    • 寻呼控制：根据 UE 当前 TA List 向相关 eNB 发起 Paging。

    SGW（Serving Gateway）

    SGW 是用户面的本地锚点（规范见 3GPP TS 23.401 §4.4.3.3）：

    • 用户面数据路由：在 eNB 与 PGW 之间转发 GTP-U 封装的用户数据包（S1-U 与 S5/S8 接口）。
    • E-RAB 内切换锚点：当 UE 在同一 SGW 覆盖范围内切换 eNB 时，SGW 无需变更，只需更新下行路径指向新 eNB。
    • 空闲态数据缓存：UE 进入 ECM-IDLE 时，SGW 缓存下行数据并触发 MME 发起寻呼。
    • 计费数据记录（CDR）：记录用户面流量用于计费。
    • 跨 3GPP 接入移动性锚点：支持 LTE ↔ 2G/3G 切换时的用户面连续性。

    PGW（PDN Gateway）

    PGW 是 UE 与外部数据网络（PDN，如 Internet）之间的接口（规范见 3GPP TS 23.401 §4.4.3.4）：

    • UE IP 地址分配：通过 DHCP 或静态配置为每个 PDN 连接分配 IPv4/IPv6 地址。
    • IP 数据包路由：将来自 Internet 的下行数据包路由到对应的 SGW/UE，将上行数据包转发到 Internet。
    • 策略与计费执行（PCEF）：与 PCRF 交互（通过 Gx 接口），执行 QoS 策略、深度包检测（DPI）、计费规则。
    • 非 3GPP 接入锚点：支持 Wi-Fi（通过 ePDG）等非 3GPP 接入的 UE 保持同一 IP 地址。

    HSS（Home Subscriber Server）

    HSS 是用户数据的统一存储中心，在 LTE 中替代 UMTS 的 HLR + AuC：

    • 存储 EPS 签约数据（允许的 APN、QoS Profile、漫游限制等）
    • 生成鉴权向量（RAND/AUTN/XRES/KASME），通过 S6a 接口（基于 Diameter 协议）提供给 MME。
    • 记录 UE 当前注册的 MME 地址。

    PCRF（Policy and Charging Rules Function）

    PCRF 负责动态策略决策（规范见 3GPP TS 23.203）：

    • 根据用户签约信息、业务类型（如 VoLTE、视频流）动态生成 PCC（Policy and Charging Control）规则，下发给 PGW 执行。
    • 支持基于流量的 QoS 控制（如保障 VoIP 的最低带宽）和计费规则（如按流量计费、按速率计费）。

    EPC 主要接口汇总

    接口 连接节点 协议 说明
    S1-MME eNB ↔ MME S1AP/SCTP/IP 控制面，NAS 信令传输
    S1-U eNB ↔ SGW GTP-U/UDP/IP 用户面，数据转发
    X2-C eNB ↔ eNB X2AP/SCTP/IP 切换信令
    X2-U eNB ↔ eNB GTP-U/UDP/IP 切换期间数据转发
    S11 MME ↔ SGW GTPv2-C/UDP/IP EPS Bearer 建立/修改/删除
    S5/S8 SGW ↔ PGW GTPv2-C + GTP-U S5 同 PLMN，S8 跨 PLMN（漫游）
    S6a MME ↔ HSS Diameter 签约数据查询、鉴权
    Gx PGW ↔ PCRF Diameter 策略与计费规则

    4.5 LTE 空口核心技术

    OFDMA / SC-FDMA

    LTE 的空口革命性突破是采用 OFDMA（Orthogonal Frequency Division Multiple Access） 下行和 SC-FDMA（Single Carrier FDMA） 上行（规范见 3GPP TS 36.211）。

    OFDMA 基本原理：

    • 将宽带信道划分为大量正交子载波（Subcarrier），子载波间隔 15 kHz（或 7.5 kHz 用于 MBMS）
    • 每个子载波带宽极窄，符号周期足够长，对多径时延扩展具有天然鲁棒性（无需 RAKE 接收机）
    • 添加 CP（Cyclic Prefix，循环前缀） 消除符号间干扰（ISI），正常 CP 长 4.7 μs，扩展 CP 长 16.7 μs
    • 频域调度：调度器可灵活将不同 RB（Resource Block）分配给不同 UE，实现多用户分集

    SC-FDMA 用于上行：SC-FDMA 本质是对 OFDMA 的预处理——先对时域信号做 DFT 扩展，再映射到子载波上，保持 OFDM 的频域调度灵活性，同时降低 PAPR（Peak-to-Average Power Ratio），节省 UE 发射功率消耗，延长电池寿命。

    LTE 支持的信道带宽：1.4 / 3 / 5 / 10 / 15 / 20 MHz，对应资源块（RB）数量：6 / 15 / 25 / 50 / 75 / 100 个。

    资源块（Resource Block）

    LTE 的最小调度单元是 RB（Resource Block）：

    • 频域：12 个子载波 × 15 kHz = 180 kHz
    • 时域：1 个时隙（slot）= 0.5 ms = 7 个 OFDM 符号（正常 CP）
    • 因此 1 个 RB 包含 84 个 RE（Resource Element）

    调度的最小粒度是 PRB Pair（1 ms 子帧 = 2 个时隙），即一次调度分配至少涉及 1 个 RB × 1 ms。

    LTE 帧结构

    LTE 定义两种帧结构（3GPP TS 36.211 §4）：

    • Type 1（FDD）：10 ms 帧包含 10 个 1 ms 子帧，每个子帧含 2 个 0.5 ms 时隙，上行和下行在不同频段同时传输。
    • Type 2（TDD）：10 ms 帧分为不同比例的上行/下行子帧，通过特殊子帧（Special Subframe）切换上下行方向，支持 7 种上下行配置比例（如 DL:UL = 2:2、3:1 等）。

    MIMO 技术

    LTE 支持多种 MIMO 模式（3GPP TS 36.211 / 36.213）：

    传输模式（TM） 技术名称 说明
    TM1 单天线端口 参考，不使用 MIMO
    TM2 发射分集（SFBC） 提升覆盖可靠性
    TM3 开环空间复用 增加吞吐量（无需 PMI 反馈）
    TM4 闭环空间复用 增加吞吐量（需 PMI/RI/CQI 反馈）
    TM5 MU-MIMO 多用户空间复用
    TM6 闭环 Rank-1 波束赋形（单流）
    TM7 单流波束赋形（非码本） 使用 UE 专属参考信号
    TM8 双流波束赋形 Release 9 新增
    TM9 最多 8 层 MIMO Release 10 新增
    TM10 CoMP Release 11，多点协作传输

    Release 8 基本配置：eNB 侧 2 或 4 天线，UE 侧 2 天线（2×2 MIMO），下行峰值：

    • 20 MHz + 64QAM + 2×2 MIMO + 码率 ≈ 1：理论最高 150 Mbps

    信道编码

    • Turbo 码：用于数据信道（PDSCH/PUSCH），编码速率 1/3，通过打孔实现不同码率，是 LTE 数据信道的核心纠错方案（相比 WCDMA 的卷积+Turbo，LTE 统一使用 Turbo 码）。
    • 卷积码：用于控制信道（PBCH、PCFICH、PHICH 等），实现相对简单。
    • CRC：用于错误检测，触发 HARQ 重传。

    HARQ（混合自动重传请求）

    LTE 的 HARQ（规范见 3GPP TS 36.212）在 MAC 层实现，具备以下特性：

    • 停等协议（Stop-and-Wait）：每个 HARQ 进程独立，发送方等待 ACK/NACK 后再决定重传
    • 下行 HARQ：最多 8 个并行进程，RTT ≈ 8 ms（1 ms TTI × 8）
    • 软合并（Soft Combining）：
      • Chase Combining（CC）：重传相同的冗余版本（RV0），接收端最大比合并
      • Incremental Redundancy（IR）：每次重传不同冗余版本（RV0→RV1→RV2→RV3），逐步提供更多编码信息，增益更大
    • HARQ 与 RLC 层的 ARQ 协同，构成两级重传机制。

    调度与 QoS

    LTE 的调度完全由 eNB 在 MAC 层实现，每个 TTI（1 ms）调度一次，支持：

    • 动态调度：每 1 ms 按需分配 RB，适用于突发性数据业务
    • SPS（Semi-Persistent Scheduling）：半静态调度，为 VoIP 等周期性小包业务预留资源，减少控制信令开销

    EPS Bearer 体系（规范见 3GPP TS 23.401）：

    • 每个 UE 建立一个默认 Bearer（Default Bearer），用于上网等非 GBR 业务
    • 专有 Bearer（Dedicated Bearer）用于 QoS 需求特殊的业务（如 VoLTE）
    • 每个 Bearer 有对应的 QCI（QoS Class Identifier），LTE 定义 9 种标准 QCI（QCI 1~9），5G 中扩展为 5QI

    主要 QCI 取值（LTE，3GPP TS 23.203）：

    QCI 资源类型 优先级 典型业务
    1 GBR 2 VoLTE 语音
    4 GBR 5 非会话视频（缓冲式）
    5 Non-GBR 1 IMS 信令
    6 Non-GBR 6 视频（TCP/YouTube）
    9 Non-GBR 9 默认承载（一般上网）

    4.6 LTE-Advanced（Release 10）

    Release 10（2011 年冻结）是首个满足 ITU IMT-Advanced 需求的版本，正式称为 4G，核心增强如下：

    载波聚合（Carrier Aggregation, CA）

    CA 允许 UE 同时在多个载波（Component Carrier, CC）上收发数据，每个 CC 最大 20 MHz，最多聚合 5 个 CC，总带宽可达 100 MHz，峰值速率理论上达 1 Gbps（DL）。

    载波类型：

    • PCC（Primary CC）：承载控制面信令，RRC 连接维护在此
    • SCC（Secondary CC）：仅承载用户面数据，可动态激活/去激活

    CA 场景：

    • Intra-band Contiguous CA：同频段连续频谱聚合（最简单）
    • Intra-band Non-Contiguous CA：同频段非连续频谱聚合
    • Inter-band CA：跨频段聚合（最复杂，UE 需支持多频段射频）

    增强型 MIMO

    • 下行最多支持 8 层（Rank 8） 空间复用（8×8 MIMO），通过 CSI-RS（信道状态信息参考信号）支持闭环反馈
    • 上行增强：支持最多 4 天线上行传输

    CoMP（Coordinated Multi-Point Transmission/Reception）

    • 多个 eNB 或小区协同传输/接收，对小区边缘用户增益显著
    • 在 Release 11 中进一步标准化

    Relay Node（中继节点）

    • 支持 Type 1 Relay（透明中继）：中继节点作为独立小区，不暴露给 UE
    • 增强小区边缘覆盖，无需光纤回传

    4.7 LTE-Advanced Pro（Release 13/14）

    Release 13（2016 年）引入 LTE-Advanced Pro（也称”4.9G”），聚焦物联网与异构网络：

    • NB-IoT（Narrowband IoT）：180 kHz 带宽，专为低速率、深覆盖、低功耗 IoT 设计（规范见 TS 36.211 NB-IoT 相关章节）
    • eMTC（Enhanced Machine Type Communication / Cat-M1）：1.4 MHz 带宽，支持语音和数据，用于可穿戴设备
    • LAA（Licensed Assisted Access）：LTE 主载波（授权频谱）+ 非授权 5 GHz 频段辅载波聚合
    • V2X（Vehicle to Everything）：车联网 PC5/Uu 接口初步标准化
    • FD-MIMO（Full Dimensional MIMO）：2D 天线阵列，支持垂直维度波束成形，为 5G Massive MIMO 奠基

    6. 关键技术演进的主线逻辑

    回顾 2G → 3G → 4G 的演进，可以归纳出以下几条清晰的技术主线：

    主线一：多址接入技术的代际革命

    • TDMA（2G） → CDMA（3G） → OFDMA（4G）

    每一代的多址接入技术变革都是底层能力的跃升：TDMA 简单但频谱效率有限；CDMA 通过扩频实现频率复用系数为 1，但受干扰限制容量上限，且 PAPR 和功率控制复杂；OFDMA 利用正交子载波和 CP 从根本上解决了多径干扰，支持灵活的频域调度，为 MIMO 提供了理想的信道估计框架，成为 4G/5G 的基石。

    主线二：调度与重传的下移趋势

    • RNC/BSC（3G） → Node-B（3.5G HSDPA） → eNB（4G）

    调度与 HARQ 重传功能从网络侧集中控制逐步下移至基站，TTI 从数十毫秒压缩至 1 ms，时延大幅降低，自适应能力显著增强。LTE 的 eNB 已集成了 UMTS 中 RNC 的全部功能，接入网架构真正实现”扁平化”。

    主线三：核心网从电路到全 IP 的演变

    • 纯电路交换（2G） → CS+PS 共存（3G） → 全 IP（4G EPS）

    GPRS 的引入开启了 PS 域与 CS 域并存的时代，但双域架构的运营维护成本高昂。LTE 彻底废除 CS 域，语音通过 VoLTE（Voice over LTE）在 IMS 架构上承载，实现真正的全 IP 网络。控制面与用户面的解耦（CUPS）也为后续 5G 的服务化架构奠定了基础。

    主线四：天线技术从单天线到 MIMO

    • 单天线（2G/3G） → 2×2 MIMO（4G R8） → 8×8 MIMO（4G R10） → Massive MIMO（5G）

    MIMO 技术通过空间复用（提速）和分集（提可靠性）成为提升频谱效率的最重要手段。LTE 标准化了完整的 MIMO 框架（SU-MIMO/MU-MIMO），并通过 CSI-RS 机制实现精确的信道状态反馈，为 5G 的 Massive MIMO（32T32R、64T64R 等）铺平了道路。

    7. 小结与展望

    从 2G GSM 到 4G LTE，移动通信走过了三十年的技术演进之路：

    • 2G 解决了”能通话”的问题，建立了数字蜂窝的基础架构和安全体系；
    • 2.5G/2.75G 通过 GPRS/EDGE 初步引入分组数据能力；
    • 3G UMTS 通过 WCDMA 实现了真正的移动宽带，软切换和 AMC 等技术奠定了现代无线系统的设计范式；
    • HSPA/HSPA+ 通过 Node-B 侧的调度革新将 3G 的实际体验大幅提升；
    • 4G LTE 以 OFDMA + 全 IP + 扁平化架构重构了移动通信系统，将时延压缩至个位数毫秒、峰值速率提升至 Gbps 量级，真正开启了移动互联网时代。

    理解这条演进脉络，是深入理解 5G NR 技术选择的前提：5G 的 Numerology（多种子载波间隔）、Massive MIMO、服务化核心网（SBA）、URLLC 低时延等设计，无不是在 LTE 的基础上针对新需求的进一步突破。
