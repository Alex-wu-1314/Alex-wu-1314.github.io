    信道编码与调制方式

    Alex Wu
    2016-04-28
    3gpp

    概述

    信道编码与调制是物理层的核心，决定了系统能在多嘈杂的无线信道上以多快的速度可靠地传输数据。4G LTE 采用 Turbo 码作为数据信道编码、咬尾卷积码（TBCC）作为控制信道编码。5G NR 在 2016 年 11 月的 3GPP RAN1 #87 次会议上做出了不同的选择：数据信道采用 LDPC 码，控制信道采用 Polar 码。这一决策的依据并非纯粹的数学美感，而是三个工程指标：单位硅面积的吞吐量、短块性能以及在极低 BLER 时的误帧率悬崖特性。

    本文涵盖的主要规范：

    规范编号 内容

    第一部分：信道编码

    1.1 编码方案总览

    3GPP TS 38.212 第 5.1 节明确规定了各信道的编码方案：

    信道/信息类型 编码方案
    DL-SCH（下行共享信道） LDPC
    UL-SCH（上行共享信道） LDPC
    PCH（寻呼信道） LDPC
    BCH（广播信道） Polar 码
    DCI（下行控制信息） Polar 码
    UCI（上行控制信息，≥12 bit） Polar 码
    UCI（上行控制信息，3~11 bit） 分组码（Reed-Muller）
    UCI（HARQ-ACK，1~2 bit） 重复码

    1.2 LDPC 码（数据信道）

    1.2.1 为什么选择 LDPC？

    LDPC（Low-Density Parity-Check，低密度奇偶校验）码由 Gallager 在 1963 年提出，因其逼近香农极限的纠错性能和天然适合并行硬件实现的特点，在 5G eMBB 场景的数据信道编码竞争中胜出：

    • 高吞吐量：基于置信传播（Belief Propagation）的迭代译码可在硬件上高度并行化，满足 5G 峰值速率（DL 20 Gbps）需求。
    • 低错误平层：相比 Turbo 码，LDPC 在高 SNR 下没有明显的错误平层（error floor）。
    • 灵活码率：通过循环缓冲区速率匹配，可灵活支持各种码率，并天然支持增量冗余 HARQ（IR-HARQ）。

    1.2.2 QC-LDPC 与准循环提升

    5G NR 采用准循环 LDPC（QC-LDPC），其核心思想是”基图 + 提升（Base Graph + Lifting）”。

    基本概念：

    与其存储一个庞大的奇偶校验矩阵 H，标准只定义一个小规模的基图（Base Graph）。基图是一个整数矩阵，每个元素要么是 -1（无连接），要么是一个移位值 v。运行时，将基图中每个 -1 替换为 Z×Z 的全零矩阵，每个移位值 v 替换为循环移位 v 列的 Z×Z 单位矩阵，即可得到实际的 H 矩阵。Z 称为提升尺寸（Lifting Size）。

    通过选择不同的 Z 值，可以得到不同码长的 LDPC 码，而无需存储多个不同大小的奇偶校验矩阵，实现了硬件的高度复用。

    1.2.3 两种基图：BG1 与 BG2

    3GPP TS 38.212 定义了两个基图，适用于不同的传输块大小（TBS）和码率场景（参见 TS 38.212 第 5.2.2 节和第 7.2.2 节）：

    参数 BG1 BG2
    基图尺寸（行 × 列） 46 × 68 42 × 52
    系统位列数 Kb 22 6 / 8 / 9 / 10（依 K 而定）
    母码长度（含提升） N = 66 × Z N = 50 × Z
    最大码块大小 8448 bit 3840 bit
    支持码率范围 1/3 ～ 22/24（约 8.89） 1/5 ～ 2/3
    适用场景 大 TBS、高码率、eMBB 高速传输 小 TBS、低码率、初始传输或控制

    注意：BG1 的前两列系统位在传输前被穿孔（punctured），实际传输的母码为 N - 2Z 比特。这是 5G NR LDPC 的重要规则之一。

    1.2.4 提升尺寸 Zc 的选择

    确定基图后，还需选择提升尺寸 Zc 以确定码长。TS 38.212 表 5.3.2-1 定义了 51 个合法的 Zc 值，从 2 到 384，组织为 8 个集合（索引 iLS = 0～7）：

    iLS Zc 集合（部分）
    0 2, 4, 8, 16, 32, 64, 128, 256
    1 3, 6, 12, 24, 48, 96, 192, 384
    2 5, 10, 20, 40, 80, 160, 320
    3 7, 14, 28, 56, 112, 224
    4 9, 18, 36, 72, 144, 288
    5 11, 22, 44, 88, 176, 352
    6 13, 26, 52, 104, 208
    7 15, 30, 60, 120, 240

    选取规则：从满足 Kb × Zc ≥ K（K 为码块信息位数）的合法 Zc 值中，选取最小值。

    1.2.6 速率匹配与冗余版本（RV）

    速率匹配的核心是循环缓冲区（Circular Buffer）机制，用于支持 HARQ 增量冗余（IR-HARQ）。

    编码后的码字被写入循环缓冲区，通过设置不同的起始读取位置定义四个冗余版本（RV0～RV3）。发射端每次传输从对应 RV 的起始位置连续读取所需数量的比特。

    冗余版本 起始位置（近似，相对 Zc） 特性
    RV0 0 自解码，首次传输，包含系统位
    RV1 ~17×Z 增量冗余（主要为奇偶校验位）
    RV2 ~33×Z 增量冗余
    RV3 ~56×Z 自解码，可单独解码

    默认重传顺序：RV0 → RV2 → RV3 → RV1。RV0 和 RV3 均为自解码 RV，可在没有前序传输的情况下独立解码，这为 gNB 处理 NACK/ACK 模糊提供了灵活性。

    Limited Buffer Rate Matching（LBRM）：当 UE 的软缓冲区不足以容纳完整循环缓冲区时，激活 LBRM，限制写入循环缓冲区的码字长度，保证 UE 可以正常处理 HARQ 合并。

    1.3 Polar 码（控制信道）

    1.3.1 为什么选择 Polar 码？

    Polar 码由 Arıkan 在 2008 年提出，是理论上第一个被严格证明能达到二进对称信道容量的线性码。5G NR 控制信道选择 Polar 码的核心原因：

    • 短块性能优异：DCI 通常仅有几十到几百比特，Polar 码在短块场景下优于 Turbo 码和 LDPC。
    • 与 CRC 的天然协同：Polar 码配合 SCL（Successive Cancellation List）译码 + CRC 辅助，可实现极低 BLER。
    • 灵活速率适配：通过冻结位选择和速率匹配，可支持广泛的码率和码长。

    1.3.2 Polar 码基本原理

    Polar 码的核心是信道极化（Channel Polarization） 现象：

    将 N 个独立的信道 W 通过递归组合，经过大量使用后，这 N 个等效信道会发生分化（极化），一部分趋向于完全可靠（容量接近 1），另一部分趋向于完全不可靠（容量接近 0）。

    编码策略：

    • 将信息位放置在可靠的子信道（较好的信道）上传输。
    • 将冻结位（frozen bits，通常为 0）放置在不可靠的子信道上。
    • 接收端利用已知的冻结位位置，通过 SC 或 SCL 算法逐步恢复信息位。

    码长：Polar 码的母码长度为 2 的幂次，即 N = 2^n。

    1.3.4 分布式 CRC（Distributed CRC）

    这是 5G NR Polar 码的一个重要创新。常规做法是将全部 CRC 位附加在信息位末尾；NR 则将 CRC 位交织插入信息位序列的内部。

    优势：SCL 译码器在搜索候选路径时，可在译码中途（而非全部完成后）进行提前 CRC 验证，一旦某条候选路径的局部 CRC 不匹配即可剪枝，显著降低译码复杂度（减少无效路径的计算），同时提升译码性能。

    具体参数：

    • PBCH：分布 3 个 CRC 位
    • PDCCH（DCI）：分布 7 个 CRC 位，CRC 多项式为 CRC24C

    1.3.5 可靠性序列（Reliability Sequence）

    TS 38.212 表 5.3.1-2 规定了长度为 1024 的全局可靠性序列，对所有信道通用。该序列是在大量仿真基础上确定的，按子信道容量从高到低排列。编码时，选择序列中前 K 个（可靠性最高的）子信道承载信息位，其余子信道为冻结位。

    1.3.6 译码：SCL + CRC 辅助

    工业实现中的标准译码方案是 CRC-Aided SCL（CA-SCL）：

    • SC（Successive Cancellation）：从左到右逐位判决，复杂度低（O(N log N)），但短块性能有限。
    • SCL（List Decoding）：维持 L 条候选路径（通常 L=8），在最后用 CRC 从 L 个候选中选择最佳，性能大幅提升。
    • 分布式 CRC 辅助：结合分布式 CRC，SCL 可在中间节点提前剪枝，进一步降低复杂度。

    PDCCH 场景中，UE 需要对同一搜索空间内的所有 DCI 候选盲检测，每次盲检均运行 SCL-L8 译码器，分布式 CRC 的提前终止特性在此场景下尤为重要。

    第二部分：调制方式

    2.1 调制基础

    数字调制将比特映射为复数符号（星座点），通过 OFDM 在频域上传输。5G NR 支持以下调制方式（TS 38.211 第 5.1 节）：

    调制方式 调制阶数 M 每符号比特数 信噪比要求（参考）
    QPSK 4 2 bit/符号 较低，抗干扰强
    16QAM 16 4 bit/符号 中等
    64QAM 64 6 bit/符号 较高
    256QAM 256 8 bit/符号 高，需优质信道

    注：TS 38.211 同时定义了 π/2-BPSK（用于 PUSCH transform precoding 场景的上行，降低 PAPR），以及 Release 17 引入实验性 1024QAM（10 bit/符号，TS 38.214 Table 5.1.3.1-4 和 CQI Table 5.2.2.1-5）。

    2.1.1 QPSK

    每个符号携带 2 bit，星座图为 4 个等间距相位点，具有最强的抗噪声能力。主要用于：

    • 覆盖边缘 / 弱信号场景
    • PDCCH 控制信道（固定使用 QPSK）
    • PRACH / PUCCH 控制信道

    Gray 编码映射（TS 38.211 表 5.1.3.1-1）：

    2.1.2 16QAM

    每个符号携带 4 bit，16 个星座点排列为 4×4 矩阵。在中等 SNR（约 15～20 dB）下提供良好的频谱效率（4 bit/符号）。

    2.1.3 64QAM

    每个符号携带 6 bit，64 个星座点排列为 8×8 矩阵。LTE 最高支持 64QAM，5G NR 在 FR1（Sub-6GHz）中广泛使用，需要约 20～25 dB SNR。

    2.1.4 256QAM

    每个符号携带 8 bit，256 个星座点排列为 16×16 矩阵。这是 5G NR eMBB 场景实现峰值速率的关键，但需要极高的信道质量（SNR > 25 dB），通常仅在近距离、LOS（视距）或毫米波场景使用。

    2.2 MCS 表格（Modulation and Coding Scheme）

    MCS 是调制阶数与信道编码码率的组合索引，由 gNB 通过 DCI 的 MCS 字段指示 UE。

    2.2.1 MCS 的作用

    MCS 索引（0～31）唯一确定：

    1. 调制阶数（Qm）：2 / 4 / 6 / 8 对应 QPSK / 16QAM / 64QAM / 256QAM
    2. 目标码率（R × 1024）：编码率归一化后的整数表示
    3. 频谱效率（SE = Qm × R，单位 bit/RE）

    MCS 索引 28～31 为保留或特殊用途（如重传标志）。

    2.3 CQI（信道质量指示）

    2.3.1 CQI 的作用

    UE 基于下行参考信号（CSI-RS 或 SSB）的测量，向 gNB 上报 CQI，反映当前下行信道质量。gNB 参考 CQI 选择合适的 MCS，形成闭环链路自适应。

    2.3.3 CQI 上报机制

    • 周期上报（Periodic CSI Reporting）：UE 按照 RRC 配置的周期在 PUCCH 上上报。
    • 非周期上报（Aperiodic CSI Reporting）：gNB 通过 DCI 触发，UE 在 PUSCH 上携带完整 CSI 报告（包括 CQI、PMI、RI 等）。
    • 半持久上报（SP-CSI）：R16 引入，平衡开销与时效性。

    2.4 链路自适应（Link Adaptation）全流程

    链路自适应是根据信道状态动态调整 MCS 的闭环机制，直接影响系统频谱效率：

    2.4.1 外环链路自适应（OLLA）

    实际网络中，gNB 还运行外环链路自适应（Outer Loop Link Adaptation） 算法：

    • 当 UE 频繁 NACK（BLER 过高）→ 降低 MCS
    • 当 UE 持续 ACK（BLER 过低）→ 提高 MCS
    • 目标：将实际 BLER 维持在设计目标附近（eMBB 约 10%，URLLC 约 0.001%）

    OLLA 是协议实现的自由度，3GPP 标准不规定具体算法，但 TS 38.214 通过 CQI 偏移量（CQI offset）提供了框架。

    2.4.2 TBS（传输块大小）计算

    3.2 PDCCH 的特殊性

    PDCCH 使用 Polar 码编码 + 固定 QPSK 调制，不进行链路自适应，调制方式和码率均为固定配置，通过聚合级别（Aggregation Level，AL = 1/2/4/8/16 CCE）来适应信道质量。AL 越高，重复度越大，覆盖能力越强，但占用资源越多。

    场景 LDPC（BG1/64QAM） LDPC（BG2/QPSK）
    目标 BLER 0.1 0.1
    典型 SINR 门限 ~20 dB ~0 dB
    频谱效率 ~3～5.5 bit/RE ~0.3～1.3 bit/RE

    小结

    对比维度 LDPC（数据信道） Polar 码（控制信道）
    规范章节 TS 38.212 第 5.3 / 7.2 节 TS 38.212 第 5.1 / 7.3 节
    应用信道 PDSCH / PUSCH / PCH PDCCH / PUCCH / BCH
    码结构 QC-LDPC，BG1 或 BG2 极化码，N = 2^n
    关键参数 Zc（8 组，51 个值） 可靠性序列（1024 位）
    译码算法 置信传播（BP，Min-Sum） SCL（L=8） + CRC
    HARQ 支持 是，RV0/1/2/3 否（Polar 不做 IR-HARQ）
    速率匹配 循环缓冲区 + 穿孔/重复 缩短（Shortening）或重复
    CRC 类型 CRC24A / CRC24B / CRC16 CRC24C（分布式插入）

    调制方面，5G NR 在 LTE 的 QPSK/16QAM/64QAM 基础上新增了 256QAM，并通过三张 MCS 表格（TS 38.214）分别覆盖 eMBB、eMBB 高速率和 URLLC 场景。链路自适应闭环由 CQI 上报 → MCS 选择 → HARQ 反馈 → OLLA 调整构成，使系统能在复杂无线环境下持续接近理论容量上限。
