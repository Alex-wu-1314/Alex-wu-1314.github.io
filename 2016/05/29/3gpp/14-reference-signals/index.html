    1. NR 参考信号总览与设计哲学

    1.1 LTE 的”常亮路灯”模式

    LTE 系统引入了小区专用参考信号（CRS，Cell-specific Reference Signal），以固定的时频图案在所有子帧中持续发射，方便任意 UE 随时进行信道估计。这种设计在低密度、低天线数时简单有效，但存在明显局限：

    • 资源浪费：CRS 占用固定 RE，即使无数据传输也照样发送；
    • 阻碍波束赋形：CRS 是小区级信号，无法针对特定 UE 施加预编码，限制了大规模 MIMO 的潜力；
    • 干扰受限：在密集组网或异构网络（HetNet）中，CRS 成为主要的小区间干扰源（CRS interference）。

    1.2 NR 的”按需点灯”哲学

    NR 彻底摒弃了 LTE 的 CRS，转向以”最小化始终在线信号”为核心的参考信号设计原则（参见 3GPP TR 38.912）。具体体现为：

    设计原则 说明
    UE 专用化 DMRS、PTRS 均是 UE 特定（UE-specific）信号，可随数据一同预编码，天然支持波束赋形
    按需发送 CSI-RS 支持周期、半持续、非周期三种触发模式；PTRS 按调度条件动态开关
    精简物理层开销 空闲态 / 无业务时几乎不发送参考信号，降低小区间干扰与终端功耗
    前置加载（Front-loaded） DMRS 放置于 slot 前段，使 UE 可在收到数据前完成信道估计，降低处理时延

    1.3 NR 参考信号全景

    NR 物理层参考信号体系可按方向与用途梳理如下：

    协议规范索引：NR 参考信号的序列生成与资源映射主要定义于 TS 38.211；信道测量与 CSI 反馈流程见 TS 38.214；高层参数配置见 TS 38.331（RRC）。

    2. DMRS（解调参考信号）

    2.1 设计目标与基本原理

    DMRS（Demodulation Reference Signal）是 NR 中最核心的参考信号，其唯一使命是为数据信道（PDSCH/PUSCH）及控制信道（PDCCH/PUCCH）提供信道估计，使接收端能够进行相干解调。

    与 LTE CRS 的根本区别：

    • DMRS 仅在调度了数据传输的资源块内发送（资源感知，resource-aware）；
    • DMRS 与数据经过相同的预编码矩阵，因此信道估计结果直接对应预编码信道，无需额外转换；
    • 支持多端口（Multi-port）传输，配合 CDM（Code Division Multiplexing）实现 MU-MIMO 端口正交。

    2.2 下行 PDSCH DMRS（TS 38.211 §7.4.1.1）

    2.2.1 映射类型（Mapping Type）

    PDSCH DMRS 的时域位置与 PDSCH 的映射类型密切相关：

    映射类型 适用场景 DMRS 起始符号（l₀） 特点
    Type A（时隙映射） 常规全时隙调度 符号 2 或 3（由高层参数 dmrs-TypeA-Position 配置） 适合中低速信道，DMRS 固定在 slot 前部
    Type B（非时隙映射） Mini-slot / 低时延 URLLC 总是在 PDSCH 分配的第一个符号 前置加载，适合 2~7 符号的短帧结构

    注：dmrs-TypeA-Position 通过 MIB 或 RRC 信令配置，取值为 pos2（符号 2）或 pos3（符号 3）。在多径时延较大时选 pos3 可改善信道估计精度。

    2.2.2 配置类型（Configuration Type）

    NR 定义了两种频域 DMRS 映射方案（见 TS 38.211 表 7.4.1.1.2-1 / 7.4.1.1.2-2）：

    Type 1：

    • 每隔一个子载波放置一个 DMRS（梳状，Comb-2）；
    • 单符号支持最多 4 个正交端口（2 个 CDM 组 × 2 个频域 OCC）；
    • 双符号支持最多 8 个正交端口。

    Type 2：

    • 每组 2 个连续子载波分配给同一 CDM 组（共 3 组），间隔为 4 个子载波；
    • 单符号支持最多 6 个正交端口（3 个 CDM 组 × 2 个 FD-OCC）；
    • 双符号支持最多 12 个正交端口；
    • 适合高阶 MU-MIMO（如 32 端口天线阵列）场景。

    选择依据：系统天线端口数与并发 MU-MIMO 层数决定 DMRS 配置类型。Type 1 开销低，Type 2 支持更多正交端口，两者通过 RRC 参数 dmrs-Type 配置。

    2.2.3 附加 DMRS 符号（Additional DMRS）

    对于高速信道（如高铁、快速移动场景），单一前置 DMRS 可能无法追踪时变信道。NR 支持在时域配置额外的 DMRS 符号：

    • 由高层参数 dmrs-AdditionalPosition 控制，取值 pos0（无额外）/ pos1 / pos2 / pos3；
    • 附加 DMRS 符号的具体位置由 TS 38.211 表 7.4.1.1.2-3（单符号）和表 7.4.1.1.2-4（双符号）定义；
    • 例如，对于 14 符号 slot + Type A + dmrs-AdditionalPosition = pos1，DMRS 出现在符号 2 和符号 11。

    单符号 vs 双符号 DMRS：

    模式 符号数 最大端口数（Type 1 / Type 2） 适用场景
    Single-symbol DMRS 1 4 / 6 轻负载、低移动性
    Double-symbol DMRS 2（连续） 8 / 12 重度 MU-MIMO、高阶调制

    2.3 上行 PUSCH DMRS（TS 38.211 §6.4.1.1）

    上行 PUSCH DMRS 设计与下行基本对称，关键差异如下：

    • 波形适配：CP-OFDM 上行与下行设计一致；DFT-s-OFDM（SC-FDMA 类）上行时，DMRS 使用低 PAPR 的 Zadoff-Chu（ZC）序列，以维持单载波特性、控制峰均比；
    • 跳频场景：使能 intra-slot frequency hopping 时，每个跳频段内均需放置 DMRS，dmrs-AdditionalPosition 最大取 pos1；
    • 符号位置：PUSCH Mapping Type A 的 DMRS 起始位置同样由 dmrs-TypeA-Position 决定（符号 2 或 3）；Type B 则从分配首符号开始。

    协议参考：PUSCH DMRS 序列生成见 TS 38.211 §6.4.1.1.1；资源映射见 §6.4.1.1.3。

    2.4 控制信道 PDCCH/PUCCH DMRS

    PDCCH DMRS（TS 38.211 §7.4.1.3）：

    • 每个控制资源集（CORESET）使用独立的 DMRS，端口 2000；
    • 使用伪随机序列（Gold 序列）生成，初始化参数依赖 CORESET 的小区 ID 及 scramblingID；
    • 支持 UE 专用波束赋形：PDCCH 的 DMRS 与数据经过同一预编码，使得 UE 可利用 PDCCH DMRS 进行精确的定向接收。

    PUCCH DMRS（TS 38.211 §6.4.1.3）：

    • 不同 PUCCH 格式（Format 0/1/2/3/4）对应不同的 DMRS 设计；
    • Format 1/3/4 使用序列跳频（sequence hopping）增强频率分集；
    • Format 0/2 较短，不包含独立 DMRS，而是通过序列本身进行隐式参考（sequence-based detection）。

    2.5 CDM 复用与天线端口容量

    NR DMRS 通过 CDM（Code Division Multiplexing） 实现多端口正交：

    • 时域 OCC（TD-OCC）：在相同时频位置，不同端口使用正交覆盖码（Orthogonal Cover Code）区分，长度为 2（双符号 DMRS 时）；
    • 频域 OCC（FD-OCC）：在同一 CDM 组内的相邻频域 RE 上，使用频域正交码区分两个端口。

    以 PDSCH DMRS Type 1 单符号为例，端口分配如下：

    CDM 组 RE（偶数子载波） 端口 0 端口 1
    CDM group 0 k = 0, 2, 4… w_f = [+1, +1] w_f = [+1, −1]
    CDM group 1 k = 1, 3, 5… w_f = [+1, +1] w_f = [+1, −1]

    最终，Type 1 单符号可支持端口 10001003（共 4 端口），Type 1 双符号可支持端口 10001007（共 8 端口）。

    2.6 准共址（QCL）关系

    DMRS 是 NR QCL（Quasi-Co-Location）框架的参考基准。根据 TS 38.214 §5.1.5，QCL 分为 4 类：

    QCL 类型 共址参数 典型应用
    QCL-TypeA 多普勒频移、多普勒扩展、平均时延、时延扩展 大尺度信道统计
    QCL-TypeB 多普勒频移、多普勒扩展 频率跟踪
    QCL-TypeC 多普勒频移、平均时延 TA 估计
    QCL-TypeD 空间接收参数（波束） 波束管理（FR2 必需）

    PDSCH DMRS 通常与 SSB 或 CSI-RS 通过 TCI-State（Transmission Configuration Indicator State）建立 QCL 关系，网络通过 DCI 指示当前激活的 TCI 状态，UE 据此确定最优接收波束。

    3. CSI-RS（信道状态信息参考信号）

    3.1 功能定位与分类

    CSI-RS（Channel State Information Reference Signal）是 NR 下行专用的多功能参考信号，定义于 TS 38.211 §7.4.1.5。其主要用途包括：

    用途 描述
    信道测量（Channel Measurement） UE 测量信道质量，上报 CQI/PMI/RI
    干扰测量（Interference Measurement） 配合 CSI-IM 测量小区间干扰
    波束管理（Beam Management） P1/P2/P3 过程中的波束测量与上报
    连接态移动性（Mobility） 作为 RRM 测量的参考，替代 LTE 的 CRS-based 测量
    时频跟踪（TRS） 特殊配置的 CSI-RS 集用于精细时频同步

    CSI-RS 按功率分为两类：

    • NZP-CSI-RS（Non-Zero-Power CSI-RS）：实际发送已知导频序列，UE 可用于信道估计与测量；
    • ZP-CSI-RS（Zero-Power CSI-RS）：对应 RE 不发送任何信号，用于速率匹配（使 PDSCH 避开这些 RE）或干扰测量。

    3.2 NZP-CSI-RS 资源配置（TS 38.211 §7.4.1.5）

    3.2.1 天线端口数

    NZP-CSI-RS 支持 1、2、4、8、12、16、24、32 个天线端口（对应端口编号 3000 起）。端口数越多，可精确表征的下行信道维度越高，但资源开销也相应增加。

    3.2.2 频域密度（Density）

    TS 38.211 表 7.4.1.5.3-1 定义了不同端口数下的频域密度选项：

    密度（D） 含义 每 RB 占用 RE 数
    3 每 RB 3 个 RE/端口 密集测量
    1 每 RB 1 个 RE/端口（梳状间隔 = 1） 标准配置
    0.5 每 2 个 RB 1 个 RE/端口（奇 RB 或偶 RB） 低开销

    注：密度 0.5 时，资源集可配置为 dot5odd（奇数 RB 组）或 dot5even（偶数 RB 组），两组互补可拼接成密度 1。

    3.2.3 时域配置

    NZP-CSI-RS 的时域配置由以下参数决定：

    • firstOFDMSymbolInTimeDomain（l₀）：资源集内第一个 OFDM 符号索引（0-based）；
    • firstOFDMSymbolInTimeDomain2（l₁）：部分双符号资源的第二个符号索引；
    • 周期（periodicityAndOffset）：支持 T_CSI-RS ∈ {4, 5, 8, 10, 16, 20, 32, 40, 64, 80, 160, 320, 640} 个 slot；相应偏移量范围为 [0, T-1]。

    3.2.4 CDM 类型

    CSI-RS 同样使用 CDM 实现多端口正交，CDM 类型包括：

    CDM 类型 描述 适用端口数
    noCDM 无 CDM，1 端口 1
    fd-CDM2 频域 2 点 OCC 2
    cdm4-FD2-TD2 频域 × 时域各 2 点，共 4 点 CDM 4
    cdm8-FD2-TD4 频域 2 点 × 时域 4 点 CDM 8

    3.3 ZP-CSI-RS：干扰测量与速率匹配

    ZP-CSI-RS 在指定 RE 上不发送任何信号，其作用有二：

    速率匹配（Rate Matching）：
    PDSCH 调度时，调度器可配置 ZP-CSI-RS，要求 UE 在速率匹配时跳过这些 RE，从而避免 PDSCH 数据与相邻小区的 NZP-CSI-RS 产生冲突（常用于协调多点传输 CoMP 场景）。通过 RRC 参数 zp-CSI-RS-ResourceToAddModList 配置，并可通过 DCI 中的 ZP CSI-RS trigger 进行非周期触发。

    干扰测量（Interference Measurement）：
    配合 CSI-IM（CSI Interference Measurement）资源使用。CSI-IM 资源与 ZP-CSI-RS 共享相同的时频位置，该位置在本小区不发送信号，UE 在此 RE 上测量到的能量即为来自其他小区的干扰加噪声（IN, Interference + Noise），用于计算 SINR 并上报 CQI。

    3.4 CSI-RS 触发机制：周期 / 半持续 / 非周期

    NR 提供灵活的 CSI-RS 调度机制（定义于 TS 38.214 §5.2.2.3）：

    每个 BWP 最多可配置：

    • 周期 NZP-CSI-RS 资源集：最多 64 个资源，每集最多 8 个资源；
    • 非周期 NZP-CSI-RS 资源集：每 BWP 最多 16 个，每集最多 8 个，DCI 触发字段最多 2 bit。

    3.5 CSI-RS 在波束管理中的应用

    在 FR2（mmWave，>6GHz）场景下，每个物理波束方向均需配置独立的 CSI-RS 资源以支持波束扫描。3GPP 将波束管理流程分为三个子过程（详细流程见本系列第 17 篇《Massive MIMO 与波束管理》）：

    过程 描述 使用信号
    P1（Beam Sweeping） gNB 侧发送端波束选择 SSB 或 CSI-RS（周期）
    P2（Beam Refinement） gNB 侧精细波束调整 CSI-RS（周期/非周期）
    P3（UE Beam Sweeping） UE 侧接收波束选择 CSI-RS

    实际部署中，网络可配置多个 CSI-RS 资源分别对应不同波束，UE 通过测量各资源的 CSI-RSRP（L1-RSRP）选出最优波束并上报，gNB 据此完成波束切换或波束精化。

    NZP-CSI-RS 与 ZP-CSI-RS 可交替配置实现波束区分：在某个时间点，仅一个方向的 CSI-RS 为 NZP，其余为 ZP，UE 可由此判断当前活跃波束。

    3.6 CSI 反馈框架

    UE 基于 CSI-RS 测量结果，通过 PUCCH 或 PUSCH 向 gNB 上报以下 CSI 内容（TS 38.214 §5.2）：

    • CQI（Channel Quality Indicator）：4 bit，反映信道质量，映射到 MCS 表；
    • PMI（Precoding Matrix Indicator）：指示推荐的预编码矩阵，分为 Type I（单面板）和 Type II（增强精度）码本；
    • RI（Rank Indicator）：推荐的传输层数（空间复用层数）；
    • LI（Layer Indicator）：指示最强的层（配合 Type II PMI）；
    • L1-RSRP / L1-SINR：波束测量结果，用于波束上报（Beam Report）。

    4. PTRS（相位跟踪参考信号）

    4.1 相位噪声问题与 PTRS 的诞生

    4.1.1 相位噪声的物理根源

    OFDM 系统的本地振荡器（Local Oscillator, LO）在实际工程中并非理想，其相位会随时间随机抖动，产生相位噪声（Phase Noise, PN）。在频域，相位噪声表现为两种效应：

    • CPE（Common Phase Error，公共相位误差）：同一 OFDM 符号内所有子载波产生相同的旋转角，等价于一个常数相位偏转；
    • ICI（Inter-Carrier Interference，子载波间干扰）：快速相位变化导致子载波间正交性破坏。

    相位噪声对系统性能的影响随载波频率的升高而急剧增加——频率每翻倍，相位噪声增加约 6 dB（功率谱密度）。在 FR2（24.25 GHz~52.6 GHz）频段，即使使用高质量振荡器，CPE 也可能造成超过 3° 的符号旋转，对 256QAM 调制下的误码率产生显著影响。

    4.1.2 PTRS 的解决方案

    3GPP R15 引入 PTRS（Phase Tracking Reference Signal，TS 38.211 §7.4.1.2），其核心思想是：

    在 OFDM 符号内插入已知导频，接收端通过比对实际接收值与理论值，估计出当前 CPE，并对所有子载波进行统一相位补偿。

    PTRS 的关键特性与相位噪声的统计特性完美匹配：

    • 时域高密度：相位噪声在时间维度具有强相关性（缓变），因此 PTRS 每符号（或每 2/4 符号）发送，以精细跟踪相位变化；
    • 频域低密度：CPE 对所有子载波影响相同，因此 PTRS 无需在每个子载波上都出现，只需稀疏插入即可。

    4.2 PTRS 资源配置（TS 38.211 §7.4.1.2 / §6.4.1.2）

    PTRS 的存在性由以下条件共同决定：

    1. 高层参数配置：PTRS-DownlinkConfig（下行）或 PTRS-UplinkConfig（上行）必须在 RRC 中配置；
    2. 调度条件满足：调度的 MCS 和带宽须超过门限（见下表），否则 PTRS 被禁止发送；
    3. SCS 与频率：低 SCS（如 15 kHz）及低频（FR1）通常不需要 PTRS；高 SCS（60/120 kHz）且 FR2 时 PTRS 收益最大。

    4.3 时域密度与频域密度

    PTRS 的密度由调度参数动态决定（TS 38.214 §5.1.6.3 / §6.2.3.1）：

    时域密度（L_PT-RS）

    时域密度 = 每隔多少个 OFDM 符号放置一次 PTRS，取值为 {1, 2, 4}，由调度 MCS 决定：

    MCS 索引（IMCS） 时域密度 L_PT-RS
    IMCS < ptrs-MCS1 PTRS 不存在（Disabled）
    ptrs-MCS1 ≤ IMCS < ptrs-MCS2 4（每 4 个符号一个）
    ptrs-MCS2 ≤ IMCS < ptrs-MCS3 2（每 2 个符号一个）
    IMCS ≥ ptrs-MCS3 1（每个符号一个）

    逻辑：MCS 越高（调制阶数越高），对相位偏转越敏感，需要更密集的 PTRS 追踪。

    频域密度（K_PT-RS）

    频域密度 = 每隔多少个 RB 放置一个 PTRS RE，取值为 {2, 4}，由调度带宽（NRB）决定：

    调度 RB 数（NRB） 频域密度 K_PT-RS
    NRB < nRB0 PTRS 不存在
    nRB0 ≤ NRB < nRB1 2（每 2 个 RB 一个）
    NRB ≥ nRB1 4（每 4 个 RB 一个）

    逻辑：调度带宽越宽，CPE 对频谱边缘的影响越大，但由于 CPE 特性一致，稀疏插入即可推断全带相位，因此带宽越大反而密度可更低（K 更大表示更稀疏）。

    频域位置

    PTRS 的精确子载波偏移由 resourceElementOffset（取值 {00, 01, 10, 11}）与调度起始 RB 共同决定（TS 38.211 表 7.4.1.2.2-1），确保在 MU-MIMO 多用户场景下不同用户的 PTRS 避免冲突。

    4.4 PTRS 端口关联与 DFT-s-OFDM 上行

    端口关联：PTRS 端口始终与一个 DMRS 端口绑定（关联关系由 DCI 指示）。接收端在完成 DMRS 信道估计后，再利用 PTRS 进行相位跟踪——这是一种”信道估计 + 相位细化”的两级处理流程。

    DFT-s-OFDM 上行 PTRS：
    当 PUSCH 使用 DFT-s-OFDM（Transform Precoding Enabled，单载波波形）时，PTRS 的处理方式完全不同：

    • PTRS 不插入频域资源块，而是在时域中插入特定的已知样本（time-domain PTRS）；
    • 这些样本在 DFT 之前插入，随 DFT 扩展至全带宽，接收端在频域提取后进行相位估计；
    • 每个 PTRS 组（PT-RS group）由 2 个连续 PTRS 样本构成，组数由 timeDensityTransformPrecoding 参数配置。

    5. TRS（时频跟踪参考信号）

    5.1 为什么需要 TRS

    NR 删除了 LTE 中始终在线的 CRS，UE 在 DRX（非连续接收）睡眠结束后唤醒时，面临时间同步和频率同步丢失的问题：

    • 时间同步：需重新估计最佳采样时刻（Timing Advance 的微调）；
    • 频率同步：本地振荡器在睡眠期间产生频率漂移，需恢复精细频率偏差（Fine Frequency Offset）估计。

    SSB 虽然提供粗略同步，但其周期长（最短 5 ms，通常 20 ms）、频域稀疏，无法满足精细跟踪需求。因此，NR 设计了 TRS——一种专门用于精细时频跟踪的 CSI-RS 配置。

    5.2 TRS 的本质：特殊配置的 NZP-CSI-RS 资源集

    TRS 没有独立的物理层实体，它是 NZP-CSI-RS 的一种特殊配置。在 RRC 信令中，通过在 NZP-CSI-RS-ResourceSet 中设置 trs-Info = true 来标识该资源集用于时频跟踪。

    协议要求（TS 38.211 结合 TS 38.214 §5.1.6.1）：

    • TRS 资源集必须包含至少 2 个单端口（1 port）NZP-CSI-RS 资源；
    • 这些资源在时域上分布于2 个连续 slot 内（FR1）或 4 个连续 slot 内（FR2）；
    • 每个 slot 内，TRS 在2 个不同 OFDM 符号上发送（时域交错分布）；
    • 频域密度通常为 3 个 RE/RB（每 4 个子载波一个），覆盖全部调度带宽。

    5.3 TRS 时频图案与跟踪原理

    以 FR1 典型 TRS 配置为例（SCS = 30 kHz，每 slot = 14 符号）：

    跟踪原理：

    • 时域跟踪（Timing）：利用同一 slot 内两个 TRS 符号的相位差，结合频域密度，通过相关算法（如互相关）估计定时偏差；
    • 频域跟踪（Frequency）：利用两个相邻 slot 的对应位置 TRS，计算符号间相位旋转，进而推算频率偏差。

    两个 slot 间隔 × 4 个 TRS RE，提供了足够的时间分集和频率分集，实现精细的时域（ns 级）和频域（Hz 级）跟踪。

    5.4 TRS 与 DRX 的协作

    TRS 的典型工作场景是 DRX 唤醒流程：

    具体步骤：

    1. gNB 以固定周期（常见配置为每 20 ms 或 40 ms 一个 TRS burst）发送 TRS 资源集；
    2. UE 在 DRX ON 开始时，先接收 TRS burst 完成本地振荡器校准；
    3. 校准完成后开始监听 PDCCH，从而避免因时频偏差导致控制信道检测失败。

    TRS 的周期由 periodicityAndOffset 参数配置，常见取值为 10、20、40、80 slot，对应 0.54 ms（SCS=30kHz 时）或 1080 ms（SCS=15kHz 时）。

    7. NR vs LTE 参考信号架构演进对比

    核心演进逻辑：

    1. 去 CRS 化：消除小区级常开参考信号，为超密集网络和节能奠定基础；
    2. UE 专用化：DMRS/PTRS 与数据同预编码，使 Massive MIMO 波束赋形无缝集成；
    3. 多功能 CSI-RS：一种信号框架支撑信道测量、波束管理、移动性、时频跟踪多个场景；
    4. 按需动态化：从”始终在线”到”事件驱动”，最小化参考信号开销。

    9. 小结

    本文系统梳理了 5G NR 四大参考信号体系，要点总结如下：

    DMRS 是 NR 最基础的参考信号，实现 UE 专用信道估计。核心设计包括：两种配置类型（Type 1/2）控制频域密度与端口正交能力；Mapping Type A/B 对应不同时延需求；附加 DMRS 符号应对高速移动场景；CDM 与 OCC 实现多端口 MU-MIMO 复用。

    CSI-RS 是 NR 的多功能下行参考信号，涵盖信道测量、干扰测量、波束管理和移动性测量。NZP-CSI-RS 支持 1~32 天线端口，三种触发模式（周期/半持续/非周期）提供灵活的测量调度能力。ZP-CSI-RS 用于速率匹配和干扰测量辅助。

    PTRS 是专为 mmWave 高频场景设计的相位噪声补偿参考信号。时域高密度、频域低密度，密度由 MCS 和调度带宽动态调整。PTRS 与 DMRS 端口绑定，实现两级信道估计与相位精化。

    TRS 是 NR 替代 LTE CRS 的精细时频跟踪参考信号，本质是特殊配置的 NZP-CSI-RS 资源集（trs-Info = true），以突发方式发送，专门服务于 DRX 唤醒同步和连续时频偏差跟踪。

    四大参考信号共同构成 NR 物理层”感知网络”，是 Massive MIMO、波束赋形、低时延传输和节能设计的共同基石。
