    一、物理信道概览

    3GPP TS 38.211 将 5G NR 物理信道划分为下行（Downlink）和上行（Uplink） 两大方向。

    下行物理信道（gNB → UE）

    信道 全称 承载内容
    PDSCH Physical Downlink Shared Channel 用户数据、寻呼、系统信息（SIB）
    PDCCH Physical Downlink Control Channel 下行控制信息（DCI），调度 UL/DL 资源
    PBCH Physical Broadcast Channel 主系统信息（MIB），随 SSB 一起发送

    上行物理信道（UE → gNB）

    信道 全称 承载内容
    PUSCH Physical Uplink Shared Channel 上行用户数据与 UCI 复用
    PUCCH Physical Uplink Control Channel 上行控制信息（UCI）：HARQ-ACK、SR、CSI
    PRACH Physical Random Access Channel 随机接入前导码（Preamble）

    说明：SRS（Sounding Reference Signal）在标准中属于上行物理信号（Physical Signal），但因其在调度、波束管理中与物理信道密切协作，本文一并详述。

    二、PDSCH — 物理下行共享信道

    2.1 功能与作用

    PDSCH（Physical Downlink Shared Channel）是 5G NR 下行数据承载的核心信道，其主要功能包括：

    • 承载 DL-SCH（Downlink Shared Channel）传输块，即用户面 IP 数据包；
    • 传输寻呼消息（PCH）；
    • 传输广播系统信息（BCH 之外的 SIB）；
    • 支持单播、组播（MBS，R17 引入）。

    2.2 时频资源分配

    频域资源分配有两种类型（TS 38.214, Section 5.1.2.2）：

    • Type 0：基于 RBG（Resource Block Group）的位图方式，支持非连续资源分配，适用于灵活调度；
    • Type 1：起始 VRB（Virtual Resource Block）+ 连续 RB 数量，用 RIV（Resource Indicator Value）编码后在 DCI 中携带，适合连续分配。

    时域资源分配由 DCI 中的”Time domain resource assignment”字段（SLIV 表索引）决定，包含（TS 38.214, Section 5.1.2.1）：

    • K0：调度时隙相对于 PDCCH 时隙的偏移；
    • S：起始符号位置（0~12）；
    • L：持续符号数（1~14）。

    PDSCH 映射类型（TS 38.211, Section 7.4.1.1.2）：

    类型 起始符号 典型用途
    Type A（slot-based） 相对于时隙起始，DMRS 在 symbol 2 或 3 常规调度，覆盖整个时隙
    Type B（mini-slot） 调度的起始符号开始，DMRS 紧贴第一个符号 低时延 URLLC，1~7 符号 mini-slot

    2.3 信道处理流程

    PDSCH 的物理层处理流程如下（TS 38.212 Section 7.2 + TS 38.211 Section 7.3.1）：

    LDPC Base Graph 选择规则（TS 38.212, Section 7.2.2）：

    • BG1：适用于较大传输块（TBS > 292 bits）和较高码率（R > 0.25），最高支持码率约 8/9；
    • BG2：适用于较小传输块（TBS ≤ 292 bits）或较低码率场景，适合 URLLC 重传。

    2.4 DMRS（解调参考信号）

    PDSCH DMRS 用于 UE 侧相干解调时的信道估计（TS 38.211, Section 7.4.1.1）。

    配置类型：

    参数 Type 1 Type 2
    频域间隔 每隔 1 个子载波 每隔 2 个子载波
    最大正交端口数 4 端口 6 端口
    CDM 组数 2 3
    典型场景 常规 MIMO（≤4 层） 大规模 MIMO（≤6 层）

    符号数配置（TS 38.214, Section 5.1.6.2）：

    • Single-symbol DMRS：DMRS 占 1 个 OFDM symbol，可通过 dmrs-AdditionalPosition 配置 0~3 个额外位置；
    • Double-symbol DMRS（front-loaded 增强）：DMRS 占连续 2 个 OFDM symbol，支持高速移动场景下更精确的信道估计。

    2.5 TCI State 与 QCL

    PDSCH 接收时，UE 通过 TCI State（Transmission Configuration Indicator State） 确定解调所用参考信号的准同址（QCL）关系（TS 38.214, Section 5.1.5）。

    QCL 类型包括：

    QCL 类型 准同址参数
    QCL-TypeA 多普勒频移、多普勒扩展、平均时延、时延扩展
    QCL-TypeB 多普勒频移、多普勒扩展
    QCL-TypeC 多普勒频移、平均时延
    QCL-TypeD 空域接收参数（Spatial Rx parameter），用于波束管理

    2.6 调制编码方案（MCS）

    MCS 由 DCI 中的 5 bit 字段指示，映射到 MCS 表（TS 38.214, Table 5.1.3.1-1/2/3）：

    • MCS Table 1：最高支持 64QAM；
    • MCS Table 2：最高支持 256QAM（高峰值速率场景）；
    • MCS Table 3：低谱效/高冗余，用于 URLLC 可靠性增强（最高 64QAM 但偏低 MCS 索引）。

    三、PDCCH — 物理下行控制信道

    3.1 功能与作用

    PDCCH（Physical Downlink Control Channel）承载 DCI（Downlink Control Information），核心功能包括：

    • 调度 PDSCH 的时频资源（DCI Format 1_0 / 1_1 / 1_2）；
    • 授权 PUSCH 发送（DCI Format 0_0 / 0_1 / 0_2）；
    • 下行功率控制命令（DCI Format 2_0 ~ 2_6）；
    • 触发 SRS 发送（DCI Format 2_3）；
    • 随机接入响应（RAR）的调度。

    3.2 DCI Format 体系

    DCI Format 用途 典型 RNTI
    0_0 PUSCH 调度（回退格式） C-RNTI, CS-RNTI, MCS-C-RNTI
    0_1 PUSCH 调度（增强格式） C-RNTI
    0_2 PUSCH 调度（R16 URLLC 紧凑格式） C-RNTI
    1_0 PDSCH 调度（回退格式） C-RNTI, P-RNTI, SI-RNTI, RA-RNTI
    1_1 PDSCH 调度（增强格式） C-RNTI
    1_2 PDSCH 调度（R16 URLLC 紧凑格式） C-RNTI
    2_0 时隙格式指示（SFI） SFI-RNTI
    2_1 预emption 指示（PI） INT-RNTI
    2_2 PUSCH/PUCCH 功率控制 TPC-PUSCH-RNTI, TPC-PUCCH-RNTI
    2_3 SRS 触发与功率控制 TPC-SRS-RNTI

    3.3 CORESET（控制资源集合）

    与 LTE 不同，NR 将 PDCCH 限制在 CORESET（Control Resource Set） 区域内传输，而非铺满整个系统带宽（TS 38.211, Section 7.3.2）。

    CORESET 定义参数：

    • 频域：通过 RB 级别的位图（最多 96 个 RB，以 6-RB 为粒度的 45 bit 位图）指定占用的 PRB 集合；
    • 时域：1、2 或 3 个连续 OFDM symbols；
    • CCE-REG 映射方式：
      • Non-interleaved：CCE 连续映射到 REG，频率局部化，适合波束赋形下的 SU 场景；
      • Interleaved：CCE 分散映射到 REG，频率分集增益，适合覆盖边缘 UE。

    3.4 搜索空间（Search Space）

    UE 通过盲检测（Blind Detection）在配置的搜索空间（Search Space）内找到属于自己的 PDCCH（TS 38.213, Section 10.1）。

    搜索空间类型：

    类型 子类型 用途
    公共搜索空间（CSS） Type0-PDCCH CSS SIB1 调度（CORESET#0）
    Type0A-PDCCH CSS 其他 SI 调度
    Type1-PDCCH CSS 随机接入响应（RAR）
    Type2-PDCCH CSS 寻呼
    Type3-PDCCH CSS PS-RNTI 节能信令
    UE 专用搜索空间（USS） — 单播 UE 专属调度

    聚合级别（Aggregation Level）：

    聚合级别 L CCE 数 RE 数（PDCCH 部分）
    1 1 54
    2 2 108
    4 4 216
    8 8 432
    16 16 864

    UE 在每个 PDCCH 监听时机（PDCCH Monitoring Occasion）内对配置的候选（PDCCH Candidates）逐一盲检，聚合级别越高，可靠性越好但占用资源越多。

    3.5 PDCCH 信道处理

    Polar Code 选择原因：Polar Code 在短码块上有逼近香农限的性能，完美匹配 DCI 这类 payload 较小（通常 40~150 bits）的控制信息编码需求。

    3.6 PDCCH 增强（R16+）

    • R16 URLLC 增强：引入 DCI Format 0_2 / 1_2 以减小 DCI 开销，同时支持更频繁的 PDCCH 监听时机（间隔可低至 1 slot）；
    • R16 PDCCH 监听节能：引入 DCI Format 2_6（不连续接收唤醒信号，WUS）；
    • R17 AI/ML 辅助：探索基于 AI 预测的 PDCCH 候选集减少方案（TR 38.843）。

    四、PBCH — 物理广播信道与 SS/PBCH Block

    4.1 SS/PBCH Block（SSB）结构

    PBCH 不单独存在，而是与 PSS（主同步信号）、SSS（辅同步信号）构成 SS/PBCH Block（SSB）（TS 38.211, Section 7.4.3）。

    SSB 时频结构：

    符号 内容
    symbol 0 PSS（Primary Synchronization Signal），占用 127 个子载波（居中），两侧各填充 8 个零子载波
    symbol 1 PBCH（子载波 0~239）+ PBCH DMRS
    symbol 2 PBCH（子载波 047 及 192239）+ SSS（子载波 56~182）
    symbol 3 PBCH（子载波 0~239）+ PBCH DMRS

    整个 SSB 占用 4 个 OFDM symbols × 240 个子载波（20 RBs）。

    SSB 内 DMRS 的位置由物理小区 ID（PCI）决定，具体为 v = N_ID^cell mod 4，DMRS 子载波位置为 {0+v, 4+v, 8+v, ...}。

    4.2 SSB 突发集合（SS Burst Set）

    在 5 ms 半帧内，网络可以发送多个 SSB，以完成波束扫描（TS 38.213, Section 4.1）：

    频率范围 最大 SSB 数 L_max 候选 SSB 时域位置规则（SCS=15kHz 举例，Case A）
    FR1，≤3 GHz 4 每半帧 2 个时隙，每时隙 symbol 2 和 8
    FR1，3~6 GHz 8 每半帧 4 个时隙，每时隙 symbol 2 和 8
    FR2，>6 GHz 64 每半帧 8 个时隙，多候选位置（SCS=120kHz）

    PBCH Payload 生成过程（TS 38.212, Section 7.1）：

    MIB 关键字段（TS 38.331, Section 6.2.2）：

    字段 含义
    systemFrameNumber SFN 高 6 bit（完整 SFN 10 bit，低 4 bit 在 PBCH payload 中携带）
    subCarrierSpacingCommon SIB1、Msg2/4 等初始接入所用的子载波间隔
    ssb-SubcarrierOffset (k_SSB) SSB 起始子载波相对于公共参考点的偏移
    dmrs-TypeA-Position PDSCH Type A DMRS 首符号位置（symbol 2 或 3）
    pdcch-ConfigSIB1 CORESET#0 和 SIB1 搜索空间配置（指向 TS 38.213 Table 13-x 的索引）
    cellBarred 小区是否禁止接入
    intraFreqReselection 同频小区重选控制

    4.4 PBCH 与 LTE PSS/SSS/PBCH 对比

    维度 LTE 5G NR
    广播信道位置 固定在 DC 附近 可灵活位于同步栅格（Synchronization Raster）上
    PCI 数量 504 1008
    广播信道编码 卷积码 Polar Code
    周期 10 ms 20 ms（SSB burst set 周期）
    波束扫描 不支持 支持（多 SSB 束）

    五、PUSCH — 物理上行共享信道

    5.1 功能与作用

    PUSCH（Physical Uplink Shared Channel）是 5G NR 上行数据传输的核心信道，承载：

    • UL-SCH 传输块（用户面上行 IP 数据）；
    • UCI（Uplink Control Information）复用：在没有 PUCCH 资源时，HARQ-ACK、CSI、SR 可复用在 PUSCH 上（TS 38.212, Section 6.3.2）。

    5.2 波形选择：CP-OFDM vs DFT-s-OFDM

    NR PUSCH 支持两种波形（TS 38.211, Section 6.3.1.4）：

    波形 全称 特性 适用场景
    CP-OFDM Cyclic Prefix OFDM 与下行一致，支持多层 MIMO 高 SNR，峰值速率
    DFT-s-OFDM DFT-Spread OFDM（SC-FDMA 升级版） PAPR 更低，覆盖增益 3~4 dB 覆盖受限 UE、边缘用户

    R15 限制：DFT-s-OFDM 模式下最多支持 1 个传输层（单流），R16 对此有进一步研究。

    5.3 时频资源分配

    频域资源分配（TS 38.214, Section 6.1.2）：

    • Type 0：RBG 位图（同 PDSCH）；
    • Type 1：RIV 编码的连续 RB 分配。

    时域资源分配：由 DCI 中 SLIV 表索引决定，参数 K2（PDCCH 时隙到 PUSCH 时隙的偏移）+ S + L。

    5.4 信道处理流程

    注：π/2-BPSK 是 NR 专为 DFT-s-OFDM 覆盖极限优化引入的调制方式，具有更低的 PAPR（TS 38.211, Section 6.3.1.2）。

    5.5 PUSCH DMRS

    PUSCH DMRS 配置与 PDSCH 类似（TS 38.211, Section 6.4.1.1）：

    • Type 1 / Type 2：配置方式同下行；
    • 映射类型 A / B：同样适用；
    • 频率跳变（Frequency Hopping）：PUSCH 支持时隙内或时隙间的频率跳变，增强频率分集，提升覆盖；
    • PT-RS（Phase Tracking Reference Signal）：主要用于 FR2（毫米波），补偿本地振荡器引入的相位噪声（CPE，Common Phase Error）。

    5.6 PUSCH 重传与 HARQ

    NR PUSCH 支持异步 HARQ（与 LTE 的同步 HARQ 不同）：

    • 重传时机由 DCI 动态指示，灵活性更高；
    • 支持码块组（CBG，Code Block Group）级别重传（TS 38.212, Section 7.3.1.1.2），仅重传出错的 CBG，提升频谱效率；
    • 上行 HARQ 进程数最多 16 个（TS 38.321, Section 5.4.2.1）。

    六、PUCCH — 物理上行控制信道

    6.1 功能与 UCI 类型

    PUCCH（Physical Uplink Control Channel）专用于传输上行控制信息（UCI），包括三类（TS 38.213, Section 9）：

    • HARQ-ACK：对 PDSCH 解码结果的 ACK（1 bit）/ NACK（1 bit）反馈，多载波聚合时可达数十 bit；
    • SR（Scheduling Request）：UE 请求上行调度资源，通常为 1 bit（有资源请求/无）；
    • CSI（Channel State Information）：信道质量指示（CQI）、预编码矩阵指示（PMI）、秩指示（RI）等，通常为 CSI Part 1 + Part 2 的多 bit 报告。

    6.2 五种 PUCCH Format

    NR 设计了 5 种 PUCCH Format，兼顾时延和可靠性需求（TS 38.211, Section 6.3.2；TS 38.213, Section 9.2）：

    Format 类型 符号数 RB 数 UCI 载荷 编码方式 典型使用场景
    0 短 PUCCH 1~2 1 ≤2 bit 序列选择（无编码） 快速 HARQ-ACK / SR，低时延
    1 长 PUCCH 4~14 1 ≤2 bit OCC + 序列扩频 覆盖受限场景的 HARQ-ACK
    2 短 PUCCH 1~2 1~16 >2 bit Polar（UCI）+ DMRS FDM 大 CSI 报告，低时延
    3 长 PUCCH 4~14 1~16 >2 bit Polar（UCI）+ DMRS TDM 大 UCI，高可靠
    4 长 PUCCH 4~14 1 >2 bit OCC 扩频 + Polar 资源受限的多用户复用

    短 PUCCH vs 长 PUCCH：

    • 短 PUCCH（Format 0/2）：1~2 个 OFDM symbol，优先满足 URLLC 低时延要求；
    • 长 PUCCH（Format 1/3/4）：4~14 个 OFDM symbol，提供时间分集，适合覆盖受限 UE 或需要高可靠传输的场景。

    6.3 Format 0 详解（序列选择机制）

    Format 0 不使用传统的编码+调制，而是通过选择不同的 ZC（Zadoff-Chu）循环移位序列来隐式携带 1~2 bit UCI（TS 38.211, Section 6.3.2.3）：

    • 1 bit HARQ-ACK：ACK 用序列索引 0，NACK 用序列索引 1（通过 TS 38.213 Table 9.2.3-3 映射）；
    • 2 bit HARQ-ACK：4 种组合映射到 4 个不同循环移位（Table 9.2.3-4）；
    • 同一时频资源上不同 UE 可使用不同循环移位实现多用户复用（MU）。

    6.4 资源配置

    UE 最多可配置 4 个 PUCCH 资源集合（TS 38.213, Section 9.2.1）：

    • 资源集合 0：用于 ≤2 bit HARQ-ACK，最多 32 个 PUCCH 资源；
    • 资源集合 1~3：用于 >2 bit UCI，各最多 8 个 PUCCH 资源。

    UE 通过 DCI 中的 3 bit 字段指示使用哪个具体资源（PUCCH resource indicator）。

    6.5 频率跳变

    PUCCH 支持时隙内频率跳变（Intra-slot FH）和时隙间频率跳变（Inter-slot FH），利用频率分集改善覆盖。时隙间跳变仅对长 PUCCH（Format 1/3/4）有效（TS 38.211, Section 6.3.2）。

    七、PRACH — 物理随机接入信道

    7.1 功能

    PRACH（Physical Random Access Channel）是 UE 向 gNB 发起随机接入的信道，用于：

    • UE 在 RRC_IDLE/INACTIVE 态发起初始接入；
    • 连接态的切换与重建；
    • 请求上行时间对齐（Timing Advance, TA）；
    • 波束失败恢复（BFR, Beam Failure Recovery）。

    每个时频 PRACH 机会（PRACH Occasion）内定义 64 个前导码（Preamble）。

    7.2 前导码序列：Zadoff-Chu 序列

    PRACH 前导码基于 Zadoff-Chu（ZC）序列（TS 38.211, Section 6.3.3）：

    $$x_u(n) = e^{-j\pi u n(n+1)/L_{RA}}, \quad n = 0, 1, …, L_{RA}-1$$

    其中 $u$ 为 ZC 根序列索引，$L_{RA}$ 为序列长度。

    ZC 序列的优良特性：

    • 恒模性：时域幅度恒定，PAPR 极低；
    • 理想循环自相关性：易于时延估计（TA 计算）；
    • 零循环互相关性（同 $L_{RA}$ 的不同根序列之间）：支持多 UE 同时接入时的碰撞检测。

    7.3 前导码格式

    7.3.1 长前导码（Long Preamble，FR1 主用）

    Format $L_{RA}$ SCS（kHz） 时域符号数 CP 长度 覆盖半径
    0 839 1.25 1×序列+GP 3168 Ts ~14 km
    1 839 1.25 2×序列+GP 21024 Ts ~77 km（超大小区）
    2 839 1.25 4×序列+GP 4688 Ts ~29 km
    3 839 5 4×序列+GP 3168 Ts ~10 km，高移动性

    长前导码 SCS（1.25 kHz / 5 kHz）与 NR 其他传输不同，使用独立的数字参数配置。

    7.3.2 短前导码（Short Preamble，FR1/FR2 通用）

    Format $L_{RA}$ 时域符号数 支持场景
    A1 139 2 常规覆盖
    A2 139 4 常规覆盖
    A3 139 6 常规覆盖
    B1 139 2 短 CP
    B2 139 4 短 CP
    B3 139 6 短 CP
    B4 139 12 大范围覆盖
    C0 139 1 极低时延
    C2 139 4 —

    短前导码 SCS 与承载 PUSCH 的 SCS 相同（FR1 可用 15/30/60 kHz，FR2 可用 60/120 kHz），支持波束扫描（同一前导码在不同接收波束下重复发送）。

    7.4 PRACH 时频资源配置

    PRACH 资源由 SIB1 中的 RACH-ConfigCommon 配置（TS 38.213, Section 8.1）：

    • **prach-ConfigurationIndex**：映射到 TS 38.211 Table 6.3.3.2-2/3/4，确定 PRACH 格式、每帧 PRACH occasion 数量及时隙/符号位置；
    • **msg1-FDM**：每个时隙内频域 PRACH occasion 的数量（1/2/4/8）；
    • **msg1-FrequencyStart**：PRACH 起始 RB 偏移（相对于 UL BWP 起始 PRB）；
    • **ssb-perRACH-OccasionAndCB-PreamblesPerSSB**：SSB 与 RACH Occasion 的映射关系，确定每个 SSB 波束对应的 CB 前导码数量。

    7.5 64 个前导码的分组

    每个 PRACH Occasion 的 64 个前导码分为（TS 38.213, Section 8.1）：

    • Group A（竞争型，Contention-Based，CB）：用于 Msg3 较小的 UE；
    • Group B（竞争型，CB）：用于 Msg3 较大的 UE（配置了 messageSizeGroupA 和 messagePowerOffsetGroupB）；
    • 专用前导码（Contention-Free，CF）：由 gNB 通过 RRC 或 PDCCH order 专门分配，用于切换、波束失败恢复等，避免碰撞。

    7.6 RA-RNTI 计算

    UE 发送 Msg1 后，用 RA-RNTI 监听 PDCCH 以接收 RAR（TS 38.213, Section 8.1）：

    $$\text{RA-RNTI} = 1 + s_{id} + 14 \cdot t_{id} + 14 \cdot 80 \cdot f_{id} + 14 \cdot 80 \cdot 8 \cdot ul_{carrier_id}$$

    其中 $s_{id}$（OFDM 符号索引，013）、$t_{id}$（时隙索引，079）、$f_{id}$（频域 PRACH Occasion 索引，0~7）共同唯一标识一个 PRACH 传输机会。

    八、SRS — 探测参考信号

    8.1 功能

    SRS（Sounding Reference Signal）是上行物理信号（非物理信道），UE 向 gNB 发送，用于（TS 38.211, Section 6.4.1.4）：

    • 上行信道探测与调度：gNB 测量上行频率选择性信道，用于频域选择性调度；
    • 链路自适应：估计上行 SINR，指导 MCS 和功率控制；
    • Massive MIMO 波束管理：结合信道互易性（TDD），gNB 利用上行 SRS 推断下行信道，生成下行预编码矩阵（PMI）；
    • UE 发送波束选择：gNB 评估不同 UE 发送波束下的信道质量，指示最优发送波束（SRS-based beam management）。

    8.2 SRS 时频资源结构

    时域（TS 38.211, Section 6.4.1.4.3）：

    • SRS 位于时隙内最后 6 个符号中的 1/2/4 个连续符号；
    • 多符号 SRS 可提供更高的频率覆盖密度，用于覆盖扩展（Coverage Extension）。

    频域（SRS 带宽配置，TS 38.211, Table 6.4.1.4.3-1）：

    • SRS 带宽通过 c_SRS（063，带宽配置索引）和 b_SRS（03，带宽层）双参数确定；
    • 支持频率跳变（Frequency Hopping）：周期性 / 半持续 SRS 可在不同子带间跳变，实现宽带信道探测。

    8.3 触发类型

    类型 触发方式 典型场景
    Periodic SRS RRC 配置固定周期（$T_{SRS}$ ∈ {1,2,4,5,8,…,2560} slots） 持续性上行信道跟踪
    Semi-Persistent SRS（SP-SRS） MAC CE 激活 / 去激活 灵活激活的周期性探测
    Aperiodic SRS DCI Format 2_3（TPC-SRS-RNTI）或 DCI 中的 SRS request 字段触发 按需立即探测（切换前、波束失败恢复等）

    8.4 SRS 天线端口与多用户正交

    UE 可在 1 / 2 / 4 个天线端口上发送 SRS（端口编号 1000~1003）。

    多 UE 在相同时频资源上发送正交 SRS，通过以下机制实现正交（TS 38.211, Section 6.4.1.4.2）：

    • 频率梳（Comb）结构：KTC ∈ {2, 4}，每 KTC 个子载波取一个，不同 UE 使用不同的梳齿偏移（comb offset）；
    • 循环移位（Cyclic Shift）：序列循环移位提供额外正交维度（每个梳最多 8 个循环移位）。

    8.5 SRS 与 CSI-RS 对比
