    1. 5G 三大场景：eMBB / URLLC / mMTC 的需求来源

    1.1 背景：ITU IMT-2020 与 3GPP 的场景定义

    在讨论具体场景之前，需要厘清需求的两个来源：ITU（国际电信联盟） 定义了 IMT-2020 的宏观愿景，而 3GPP 则将其转化为可工程化实现的技术规范。

    ITU 层面：ITU-R M.2083（2015 年）正式确立 IMT-2020 框架，将 5G 的应用场景归纳为三大类，并给出八项关键性能指标（KPI）：峰值速率、用户体验速率、频谱效率、移动性、时延、连接密度、网络能效、区域流量密度。

    3GPP 层面：在研究课题 FS_SMARTER（TR 22.891，”面向新服务和市场技术赋能的可行性研究”）中，3GPP 梳理了超过 70 个具体用例，并将其汇聚为 eMBB、URLLC、mMTC 三大类别。随后，三大场景分别在 TR 22.861（eMBB）、TR 22.862（URLLC/CriC）、TR 22.863（mMTC）中深化研究，最终汇总至规范性文档 TS 22.261（5G 系统服务需求）。

    关键区别：与 4G 系统试图用”一套方案覆盖所有需求”不同，TS 22.261 明确指出，5G 系统旨在为不同服务、不同流量负载、不同用户群体提供优化的差异化支持。

    1.2 eMBB（增强移动宽带）

    需求来源

    eMBB 的驱动力来自人类对多媒体内容的持续渴求：

    • 视频流媒体爆炸：4K/8K 视频、VR/AR 沉浸体验对下行带宽的需求急剧攀升；
    • 高密度场景：体育场、演唱会、地铁站等高密度公共场所，用户密度极大，LTE 容量捉襟见肘；
    • 云端化趋势：云游戏、远程桌面、企业协作等业务要求稳定的高速率连接；
    • 固定无线接入（FWA）：替代光纤宽带的”最后一公里”解决方案。

    TS 22.261 将 eMBB 场景细分为两类：

    子场景 特征 典型部署
    热点（Hotspot） 高用户密度、极高容量、低移动性 室内办公、体育场、购物中心
    广域覆盖（Wide Area） 无缝覆盖、中高移动性、显著改善的用户速率 郊区、高速公路、高铁

    关键 KPI（来自 TS 22.261 §7.1）

    • 峰值下行速率：20 Gbps（ITU IMT-2020 要求）
    • 峰值上行速率：10 Gbps
    • 用户体验下行速率（室内热点）：1 Gbps
    • 用户体验下行速率（广域覆盖）：50 Mbps
    • 峰值频谱效率（下行）：30 bps/Hz（8 层空分复用）
    • 区域流量密度：室内热点可达 15 Tbps/km²，对应 250,000 用户/km²
    • 移动性：支持高达 500 km/h（高铁场景）

    典型应用

    1.3 URLLC（超可靠低时延通信）

    需求来源

    URLLC 的驱动力来自工业互联网、公共安全和智能交通领域，这些领域中通信失败的代价不仅是用户体验下降，而是人身安全与工业损失。

    3GPP TR 22.862（CriC/URLLC 研究报告）及后续的 TR 22.804（垂直域通信自动化研究）梳理了核心驱动力：

    • 工业 4.0：柔性制造中的机械臂协同控制、AGV 调度要求控制环路时延极低；
    • 远程操控/遥控驾驶：操作人员与远端机器之间的往返时延必须在感知阈值（约 10 ms）以内；
    • V2X（车联网）：车辆碰撞预警、交叉路口协同需要端到端时延 <10 ms、可靠性 >99.999%；
    • 医疗（远程手术）：外科机器人的精确控制要求极低抖动和极高可靠性；
    • 电网保护：配电网差动保护继电器的动作时延要求在 8 ms 以内。

    关键 KPI（来自 TS 22.261 §7.2）

    指标 数值 备注
    用户面时延（UL） 0.5 ms 触觉交互场景
    用户面时延（DL） 0.5 ms 触觉交互场景
    用户面时延（eMBB 参考值） 4 ms DL / UL
    可靠性 99.9999%（六个九） 1 ms 内数据包投递成功率
    端到端时延（V2X） ≤ 3 ms 高优先级场景
    可用性 >99.9999% 工业控制

    注意：上述 0.5 ms 为空口单向时延（one-way radio latency），不含核心网和应用层处理时延。实际端到端时延需综合考量网络拓扑。

    技术挑战

    URLLC 在物理层带来了根本性挑战：

    • 短包传输：传统 FEC 码针对长码块优化，短包情况下编码增益不足；
    • 小时隙（Mini-Slot）调度：无法等待整个 slot 才传输，需要符号级别的抢占调度；
    • 重传代价：HARQ 重传会引入 RTT 时延，与低时延目标矛盾；
    • 可靠性与资源效率的权衡：高冗余编码消耗大量频谱资源。

    1.4 mMTC（海量机器类通信）

    需求来源

    IoT 设备的爆炸式增长是 mMTC 的核心驱动力。Ericsson、Cisco 等机构预测，到 2030 年全球连接物联网设备将超过数百亿台。这类设备的特点与 eMBB 截然相反：

    • 传输数据量极小（传感器读数、状态上报）；
    • 对时延不敏感（周期性上报间隔分钟/小时级）；
    • 对功耗极度敏感（要求电池寿命 10 年以上）；
    • 成本要求极低（模组成本需降至数美元量级）；
    • 部署环境恶劣（地下管道、偏远农村、建筑内深层覆盖）。

    关键 KPI（来自 TS 22.261 §7 & TR 22.863）

    指标 数值
    连接密度 1,000,000 设备/km²
    电池寿命 ≥ 10 年（AA 电池）
    覆盖增强 相比 LTE Cat-1 额外 20 dB MCL 改善
    数据速率 低（kbps 量级即可满足）
    时延容忍 秒~小时级均可接受

    与 4G 的关系：从部署现实来看，5G Release 15/16 主要聚焦 eMBB 和 URLLC。mMTC 在 5G NR 中由 3GPP 早期（R13）就已引入的 NB-IoT 和 LTE-M（eMTC） 承担，二者被纳入 5G 体系以满足 IMT-2020 要求。面向中高端 IoT 需求（介于 eMBB 与 mMTC 之间），R17 引入了 RedCap（Reduced Capability）。

    来源：ITU-R M.2083（IMT-2020 愿景）及 3GPP TS 22.261

    1.6 场景间的设计张力

    三大场景的 KPI 需求存在内在冲突，对空口设计构成根本约束：

    • eMBB vs URLLC：eMBB 希望大带宽、高阶调制（256QAM）、长码块以提升吞吐量；URLLC 需要小时隙、短包、低码率强编码和快速 HARQ 反馈，两者在资源分配上相互竞争。
    • eMBB vs mMTC：eMBB 需要复杂的大规模 MIMO 与频繁的信令交互；mMTC 设备要求极简协议栈和深度覆盖（MCL 164 dBm 量级）。
    • URLLC vs mMTC：URLLC 需要冗余资源保障可靠性；mMTC 要求极高资源利用效率。

    正是为了在同一套空口上同时满足三大场景，5G NR 才引入了 Numerology（灵活子载波间距）、Mini-Slot、网络切片、灵活帧结构等一系列革命性设计，这些内容将在 §3 详细展开。

    2. NSA（Option 3x）vs SA（Option 2）组网策略与部署现状

    2.1 3GPP 定义的 8 种部署选项全览

    在 3GPP RAN 第 72 次全会上，针对 5G 网络部署定义了 8 种架构选项（Option 1~8），按照核心网类型（EPC vs 5GC）和接入节点类型（eNB vs gNB/ng-eNB）进行组合：

    规范依据：NSA 多无线双连接（MR-DC）架构详见 3GPP TS 37.340；SA 架构详见 3GPP TS 38.300。

    Option 6 和 Option 8 因缺乏实际部署价值，在标准化阶段就被业界放弃。Option 3x（NSA）和 Option 2（SA） 已成为行业共识的两种主要商用部署路径。

    2.2 NSA Option 3 / 3a / 3x 详解

    基本架构

    NSA Option 3 系列均属于 EN-DC（E-UTRA-NR Dual Connectivity） 架构，核心特征是：

    • 核心网：沿用 4G EPC（Evolved Packet Core），即 SGW/PGW 不变；
    • 主节点（Master Node，MN）：LTE eNB，负责全部控制面（S1-MME、X2-C）；
    • 辅节点（Secondary Node，SN）：5G NR 基站（en-gNB，”en” 表示连接 EPC 的 gNB），仅承载用户面数据；
    • UE：同时与 eNB 和 en-gNB 保持连接。

    控制面路径：UE ↔ eNB ↔ EPC（via S1-MME），NR 侧无直接控制面信令到核心网。

    Option 3 / 3a / 3x 的区别（仅用户面路径不同）

    三种子选项的差异完全体现在下行数据的路由方式：

    选项 用户面路由（下行） 特点
    Option 3 EPC → eNB → 分流至 en-gNB eNB 作为数据锚点，en-gNB 数据来自 eNB（X2-U 接口分流）；对 EPC 无修改，但 X2 接口压力大
    Option 3a EPC → eNB（独立）+ EPC → en-gNB（独立） EPC 直接向 en-gNB 建立独立 S1-U 隧道；减少 X2 压力，但 EPC 需支持双流分发
    Option 3x EPC → en-gNB → 分流至 eNB en-gNB 作为用户面锚点；充分利用 NR 大带宽承载主数据流，LTE 路径分担余量；对 EPC 修改最少，是业界部署最广泛的选项

    Option 3x 胜出的原因：5G NR 频段（尤其是 3.5 GHz 的 n78 频段）拥有更大带宽（100 MHz），以 en-gNB 作为用户面锚点可最大化利用 NR 吞吐量；同时对 EPC 核心网改动最小，部署成本最低。

    NSA 的优势与局限

    优势：

    • 可复用现有 LTE 无线站点和 EPC，5G 无线网可快速叠加部署；
    • eNB 提供覆盖锚点，解决早期 NR 覆盖不连续问题；
    • 核心网无需立即升级至 5GC，降低初期投资风险；
    • 可通过软件升级逐步演进至 SA。

    局限：

    • 无法充分发挥 5GC 的网络切片、URLLC 等核心能力（受限于 EPC）；
    • 控制面依赖 LTE，NR 切换受 4G 网络质量制约；
    • 无法实现独立的 5G 注册（UE 必须先有 LTE 锚点）；
    • 端到端时延仍受 EPC 影响，URLLC 场景支持能力不足；
    • 对 NB-IoT/eMTC 等 mMTC 能力无增益。

    2.3 SA Option 2 详解

    基本架构

    SA Option 2 是 5G 的目标架构，完全独立运行于 5G 体系之上：

    • 核心网：全新 5GC（5G Core Network），即 AMF、SMF、UPF 等服务化网元；
    • 接入网：5G NR gNB（Next Generation Node B），通过 NG 接口连接 5GC；
    • UE：直接注册至 5GC，无需 LTE 锚点；
    • 控制面：UE ↔ gNB ↔ AMF（via N2）；
    • 用户面：UE ↔ gNB ↔ UPF（via N3）。

    SA 的核心能力

    SA 是 5G 差异化能力的真正载体：

    • 网络切片（Network Slicing）：通过 NSSF（网络切片选择功能）为不同业务分配独立的虚拟网络资源（S-NSSAI），eMBB/URLLC/mMTC 可共享物理基础设施但逻辑隔离；
    • 服务化架构（SBA）：5GC 网元通过 HTTP/2 + JSON 的 SBI（Service Based Interface）互通，支持微服务化部署；
    • MEC（多接入边缘计算）：UPF 可下沉至边缘，结合 5GC 的 ULCL（上行分类器）实现本地流量卸载，实现真正意义上的低时延；
    • 独立的 5G 注册和移动性管理：AMF 独立处理 5G NAS 信令，支持 RRC_INACTIVE 态等 5G 专属优化；
    • 完整 URLLC 支持：5GC 的 QoS 框架（5QI）与 NR 空口的 mini-slot 协同，才能实现端到端 1 ms 时延；
    • 独立语音（VoNR）：原生支持 IMS over 5G，无需 EPS Fallback 回退。

    2.4 NSA vs SA 深度对比

    维度 NSA Option 3x SA Option 2
    核心网 4G EPC 5GC（AMF/SMF/UPF）
    控制面锚点 LTE eNB 5G gNB
    UE 注册 4G MME 5G AMF
    初始部署成本 低（复用 LTE 基础设施） 高（需全新 5GC）
    5G 独立覆盖要求 不需要（LTE 托底） 需要（gNB 独立覆盖）
    网络切片 不支持 完整支持
    URLLC 端到端 受限于 EPC 完整支持
    mMTC（NB-IoT 增强） 部分支持 完整支持
    MEC 本地卸载 受限 UPF 灵活下沉
    VoNR （通常需 EPS Fallback） 原生支持
    5G 专属移动性优化 （依赖 4G HO 机制） CHO、DAPS 等
    演进路径 可升级至 Option 2/4/7 5G 目标架构
    商用时间 较早（2019 年起） 较晚（2020 年起逐步成熟）

    2.5 其他 Option（4 / 5 / 7）简介

    • Option 4（gNB 主 + ng-eNB 辅 + 5GC）：5GC 核心网，5G gNB 作为主节点，LTE ng-eNB 作为辅节点提供覆盖补充。适合 5G 覆盖已足够但仍需 LTE 容量补充的过渡期，目前较少商用。
    • Option 5（ng-eNB + 5GC）：即”增强型 LTE”（eLTE），LTE 无线 + 5GC 核心网，是 4G 网络对接 5GC 的升级路径，可使现有 LTE 用户享受网络切片等 5GC 能力。
    • Option 7（ng-eNB 主 + gNB 辅 + 5GC）：5GC 核心网，LTE 作为控制面主节点，NR 辅助。目前商用较少，主要作为 Option 5 向 Option 2 演进的中间态。

    2.6 全球部署现状与演进路径

    部署现状（截至本文写作时）：

    全球主要运营商最初大多选择 Option 3x（NSA） 作为 5G 首发方案，理由很简单：以最小成本最快速度推出 5G 服务（主要是 eMBB），吸引用户从 4G 升级。

    随后，随着 5GC 产品成熟和 5G 专网、网络切片、工业互联网等场景需求兴起，Option 2（SA） 加速商用。中国（三大运营商）、美国（Dish Network）、德国电信、SK Telecom 等已建设或正在大规模建设 SA 网络。

    典型演进路径：

    GSMA 的实施指南指出，NSA 生态系统成熟度领先 SA 约 6 个月，运营商可根据自身网络情况选择跨越式直接 SA 或渐进式 NSA→SA 演进。

    3. 5G NR 相对 LTE 的核心改进点

    5G NR 的设计并非 LTE 的线性升级，而是在帧结构、编码、天线、架构等多个维度进行了系统性重构，以同时满足 eMBB、URLLC、mMTC 三大场景的迥异需求。以下分维度深入分析。

    3.1 帧结构与 Numerology 的革命性变化

    LTE 的局限：固定的 Numerology

    LTE 仅支持单一子载波间距 15 kHz，每个时隙固定 0.5 ms（7 个 OFDM 符号，使用正常 CP），最小调度单元（TTI）为 1 ms（1 个子帧）。这种固定设计在覆盖 eMBB 时尚可，但对于 URLLC（需要 0.5 ms 时延）和毫米波（高相噪要求更大子载波间距）则无能为力。

    5G NR 的解决方案：灵活 Numerology

    3GPP TS 38.211 定义了 5G NR 的 Numerology 体系，引入参数 μ（mu），子载波间距按 $\Delta f = 2^\mu \times 15$ kHz 计算：

    μ 子载波间距 每时隙符号数（Normal CP） 每子帧时隙数 时隙时长 主要应用场景
    0 15 kHz 14 1 1 ms FR1 广域覆盖（与 LTE 兼容）
    1 30 kHz 14 2 0.5 ms FR1 城区 eMBB 主流选项
    2 60 kHz 14（12 扩展 CP） 4 0.25 ms FR1 低时延 / FR2 部分场景
    3 120 kHz 14 8 0.125 ms FR2（毫米波）eMBB
    4 240 kHz 14 16 62.5 μs FR2 同步信号（SSB）仅用

    来源：3GPP TS 38.211 Table 4.2-1（V15.3.0）

    关键设计理念：无论 μ 取何值，帧长固定 10 ms，子帧固定 1 ms，变化的是每个子帧内包含的时隙数（2^μ 个）。这确保了不同 Numerology 之间可以时域对齐，便于在同一载波上混合使用。

    Mini-Slot（微时隙）

    除完整时隙（14 符号）外，NR 还支持 Mini-Slot 调度，即仅使用 2、4 或 7 个 OFDM 符号进行传输。Mini-Slot 是 URLLC 的关键使能技术：

    • 无需等待时隙边界，可在任意符号位置开始传输；
    • 结合 μ=2（60 kHz，0.25 ms/slot），Mini-Slot 可将空口等待时延压缩至符号级（约几十微秒）；
    • 支持 URLLC 业务抢占已调度的 eMBB 数据（通过 DCI format 2_1 发送抢占指示）。

    TDD 灵活帧结构

    NR 的 TDD 帧结构不再像 LTE 那样使用固定的上下行配比（如 TDD Config 2），而是通过 Slot Format Indicator（SFI） 在每个时隙内灵活指定每个符号的方向（Downlink / Uplink / Flexible），由 RRC 配置半静态部分，DCI 指示动态部分，最大化频谱利用效率。

    3.2 信道编码：从 Turbo 到 LDPC + Polar

    LTE 的方案：Turbo 码一统天下

    LTE 对数据信道（PDSCH/PUSCH）和控制信道（PDCCH）均使用 Turbo 码（咬尾卷积码用于 PBCH 等少量场景）。Turbo 码在 LTE 时代性能已属优异，但其缺点在 5G 高速率场景暴露无遗：

    • 解码复杂度随码块长度急剧增加，在 Gbps 级吞吐量下，Turbo 解码器的硬件面积和功耗不可接受；
    • 并行化能力有限，Turbo 解码本质上是串行迭代过程，难以在硬件上充分展开；
    • 低码率下的增益依赖重复编码（码率 < 1/3），效率不高。

    5G NR 的方案：LDPC + Polar 双引擎

    5G NR 摒弃了 Turbo 码，在数据信道与控制信道上采用不同的编码策略，实现”按需分配最优编码”：

    ① 数据信道（PDSCH / PUSCH）：LDPC 码

    LDPC（Low-Density Parity-Check，低密度奇偶校验）码由 Gallager 于 1963 年提出，2016 年在 3GPP RAN1 #86 会议上被选为 5G 数据信道编码方案（华为提案的 Polar 与 LDPC 均在该会议最终确定分工）。

    NR LDPC 的核心设计：

    • 定义了两种基图（Base Graph）：BG1（适用于大码块，长度 > 3840 bit）和 BG2（适用于小码块），涵盖从高码率（约 8/9）到低码率（约 1/3）的范围；
    • 基于准循环（QC-LDPC） 结构，天然支持硬件并行化（向量处理器、GPU 加速）；
    • 奇偶校验矩阵可延伸至比 Turbo 码更低的码率，在低码率下的编码增益更好，有利于高可靠性场景；
    • 支持 HARQ 增量冗余（IR）：不同轮次重传发送不同冗余版本（RV0~RV3），软合并累积接收能量，无需全部解码即可显著提升成功概率。

    ② 控制信道（PDCCH / PUCCH / PBCH）：Polar 码

    Polar 码由土耳其学者 Erdal Arıkan 于 2008 年提出，是理论上首个被证明可达信道容量的编码方案。在 3GPP RAN1 #87 会议上击败 LDPC 和 Tail-Biting 卷积码，成为 5G 控制信道的编码方案。

    NR Polar 码的核心特征：

    • 对短码块性能出色（控制信息通常仅几十到几百比特）；
    • 通过 CRC 辅助的列表译码（CA-List Decoding） 显著提升短块性能；
    • 编码结构简洁，便于硬件低功耗实现；
    • 相比 LTE PDCCH 使用的咬尾卷积码，在相同码长下可获得约 0.5~2 dB 增益。

    3.3 多天线技术：Massive MIMO 与波束管理

    LTE 的天线能力

    LTE 支持最多 8 发 8 收天线（通过 UE Category 8 支持 8 层空分复用），实际部署中通常为 2T2R 或 4T4R。LTE 对 Beamforming 的支持有限，主要是基于码本的波束形成，天线物理尺寸和重量限制了大规模部署。

    5G NR 的 Massive MIMO

    NR 对天线能力的定义在 3GPP TS 38.211 / TS 38.214 中规范，在 TR 38.913（NR 场景和需求）中提出性能目标：

    • 天线规模大幅扩展：NR 在下行支持最多 256 天线端口，上行支持 32 端口；典型的 64T64R 有源天线单元（AAU）在 3.5 GHz 频段已广泛部署；
    • 下行峰值频谱效率：30 bps/Hz（8 层 SU-MIMO），相比 LTE 的 16 bps/Hz 近乎翻倍；
    • 波束赋形（Beamforming）：大量天线阵列可在三维空间（方位角 + 俯仰角）形成高增益窄波束，将能量精确指向目标 UE，而非广播辐射；
    • MU-MIMO（多用户 MIMO）：通过空间隔离，同时在同一时频资源上服务多个 UE，显著提升系统级频谱效率；
    • 3D-MIMO / FD-MIMO：在垂直维度（仰俯角）也进行波束控制，实现立体空间复用。

    波束管理（Beam Management，BM）

    毫米波（FR2，24.25 GHz~52.6 GHz）的高路径损耗要求窄波束才能维持链路预算，因此 NR 引入了完整的波束管理（BM） 框架（3GPP TS 38.214 §5.1.4 / §6.1.4）：

    • P1（波束扫描）：gNB 周期性发送多个 SSB/CSI-RS 波束，覆盖不同空间方向；
    • P2（波束测量）：UE 测量各波束的 RSRP/RSRQ，上报最优波束；
    • P3（波束精化）：在粗选波束的基础上，通过 CSI-RS 进行更精细的波束对准；
    • 波束失败恢复（BFR）：快速检测波束失败并触发候选波束恢复，保证通信连续性；
    • L1-RSRP 上报：与 RRC 层 A/B 事件不同，波束管理依托物理层 L1-RSRP 测量实现快速切换（毫秒级）。

    FR1 vs FR2 对比

    特性 FR1（Sub-6 GHz，< 7.125 GHz） FR2（毫米波，24.25~52.6 GHz）
    典型频段 n78（3.5 GHz）、n41（2.5 GHz） n257（28 GHz）、n258（26 GHz）
    最大信道带宽 100 MHz 400 MHz
    覆盖距离 较远（公里级） 较近（数百米）
    穿透损耗 较小 极高（玻璃、墙体可造成 10~40 dB 损耗）
    Massive MIMO 64T64R（主流） 可达 256T（相控阵）
    波束赋形必要性 推荐 必须（路径损耗补偿）
    主要应用 宏覆盖 + eMBB 热点 eMBB、固定无线接入（FWA）

    3.4 波形与上行增强

    下行波形：CP-OFDM（与 LTE 一致）

    NR 下行沿用 CP-OFDM（循环前缀正交频分复用），但通过 Numerology 支持更大带宽（单载波最大 100 MHz @FR1，400 MHz @FR2）。

    上行波形：CP-OFDM + DFT-s-OFDM（新增）

    LTE 上行使用 SC-FDMA（单载波频分多址，即 DFT-s-OFDM 的一种形式），峰均比（PAPR）低，对 UE 功放友好。

    NR 上行同时支持：

    • CP-OFDM：支持上行 MIMO 多流传输（最大 4 层），适合近点 UE；
    • DFT-s-OFDM：延续 LTE SC-FDMA 的低 PAPR 特性，适合覆盖边缘 UE（功放效率优化）。

    两种波形可由 gNB 动态调度选择，在吞吐量与覆盖之间灵活权衡。

    上行补充信道：SUL（补充上行）

    NR 引入 SUL（Supplementary Uplink），允许在一个低频段（如 2.1 GHz）部署上行覆盖补充，与 TDD 高频段（如 3.5 GHz）的下行共同工作，解决 TDD 系统上行覆盖不足的问题（因 TDD 上下行共享带宽，上行时隙比例通常较少）。

    3.5 架构改进：CU/DU 分离与服务化核心网

    NG-RAN：CU/DU/RU 功能拆分

    LTE eNB 是单体架构，基带与射频一体化。5G NR 引入了 gNB 内部的功能拆分（3GPP TS 38.401 定义）：

    • gNB-CU（集中单元）：处理 RRC / PDCP / SDAP，可进一步拆分为 CU-CP（控制面）和 CU-UP（用户面），通过 E1 接口互连；
    • gNB-DU（分布单元）：处理 RLC / MAC / 物理层高层，通过 F1 接口（F1-C 控制面 / F1-U 用户面）与 CU 互连；
    • RU（射频单元）：实现物理层底层（FFT/iFFT、前向纠错等）及射频，通过前传（Fronthaul） 与 DU 相连（O-RAN 定义了开放前传接口 O-RAN FH）。

    CU/DU 分离的价值：CU 云化（部署于数据中心），DU 靠近天线部署，在集中化处理（降成本）与低时延（分布化）之间取得平衡。

    5GC：服务化架构（SBA）

    LTE 核心网 EPC 采用”点对点”接口（S11、S5/S8 等 GTP-C），而 5GC 引入 SBA（Service Based Architecture）：

    • 所有网元通过 SBI（Service Based Interface） 注册到 NRF（NF Repository Function） 进行服务发现；
    • 接口协议使用 HTTP/2 + JSON（RESTful API），替代 GTP-C + Diameter；
    • 网元可按微服务粒度独立扩缩容；
    • 控制面（AMF/SMF/PCF/NSSF 等）与用户面（UPF）彻底分离（CUPS，Control and User Plane Separation），UPF 可灵活下沉。

    3.6 其他关键改进

    带宽部分（BWP，Bandwidth Part）

    NR 引入 BWP 概念（TS 38.213 / 38.214），允许 UE 仅激活载波带宽的一个子集（如在 100 MHz 载波上仅激活 20 MHz BWP）：

    • 省电：UE 仅需监听激活的 BWP，降低基带处理功耗；
    • 灵活性：不同 BWP 可配置不同 Numerology，同一载波上同时服务多场景；
    • 前向兼容：为未来新信号预留资源，不影响当前 UE。

    免调度上行（Configured Grant / Type 1 & 2）

    相比 LTE 仅支持 SPS（半静态调度），NR 进一步增强了配置授权（CG） 机制，允许 URLLC 业务在不经历 SR→BSR→UL Grant 的完整调度流程下直接发送数据，大幅减少上行时延。

    HARQ 增强

    特性 LTE 5G NR
    HARQ 进程数 最多 8 个 最多 16 个
    异步 HARQ 仅 FDD 支持 TDD 和 FDD 均支持异步 HARQ
    HARQ 码本 静态 动态 / 半静态码本，URLLC 可配置禁用重传（减少时延）
    CBG（码块组）重传 不支持 支持，减少不必要的完整 TB 重传

    参考信号重构

    LTE 使用 CRS（小区专属参考信号）连续发送，即便无数据也持续占用资源（”always-on” 信号），造成干扰和功耗浪费。NR 彻底重构参考信号体系：

    • DMRS（解调参考信号）：前置（Front-loaded）设计，尽量靠近数据符号位置，降低信道估计误差，且仅在有数据时发送；
    • CSI-RS（信道状态信息参考信号）：周期/非周期/半持续多种方式，按需配置；
    • PTRS（相位追踪参考信号）：专为毫米波设计，补偿高频段的相位噪声；
    • TRS（追踪参考信号）：辅助 UE 时频追踪，降低功耗；
    • SRS（探测参考信号）：上行，用于信道估计和 TDD 上下行互易性校正，支持 Massive MIMO 的波束成形权值计算。

    初始接入：SSB 替代 CRS + PSS/SSS

    LTE 的同步信号（PSS/SSS）和 PBCH 分散在不同时频位置，NR 将其整合为 SS/PBCH Block（SSB）：PSS（127 子载波）+ SSS（127 子载波）+ PBCH（240 子载波）封装在连续的 4 个 OFDM 符号内，支持波束扫描（多个 SSB 可代表不同波束方向），UE 搜索更高效。

    总结

    本文系统梳理了 5G NR 三大核心主题：

    三大场景（§6） 是 5G 存在的根本理由。eMBB 追求极致带宽体验（20 Gbps 峰值速率），URLLC 追求极限可靠低时延（0.5 ms、六个九可靠性），mMTC 追求海量连接低功耗（100 万设备/km²、10 年电池寿命）。三者 KPI 的根本冲突，驱动了 5G NR 全系统的设计哲学——灵活、可配置、场景驱动。

    NSA vs SA（§7） 是 5G 从概念到商用的现实路径选择。Option 3x 以最小代价快速推出 5G 服务，Option 2 则是承载全部 5G 能力（切片、MEC、URLLC、VoNR）的目标架构。全球运营商正沿着 NSA→SA 或直接 SA 的路径快速演进。

    NR 核心改进（§8） 则是工程实现三大场景的技术底座：灵活 Numerology 同时支持 eMBB 的高效率与 URLLC 的低时延；LDPC+Polar 替换 Turbo 码大幅提升编码效率；Massive MIMO + 波束管理提升空口容量；CU/DU 分离与 5GC SBA 赋予网络云化和弹性扩展能力。
