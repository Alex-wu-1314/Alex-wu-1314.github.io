    1. 概述

    GCC 提供了数量庞大的隐式声明内建函数（implicitly-declared built-in functions），这些函数通常由编译器直接内联展开（inline）。内建函数的主要用途包括：

    • 优化目的：对应标准 C 库函数，编译器可以将其优化为更高效的指令序列。
    • 暴露底层功能：实现某些库函数或语言特性所需的低层操作。
    • 目标机器特定指令：不需要内联汇编即可访问特定 CPU 指令。
    • 运行时错误检查：支持各种边界检查和安全检查机制。

    命名约定

    • 绝大多数内建函数以 __builtin_ 为前缀。
    • 少数函数不遵循此约定（如 __sync_* 系列）。
    • 除特别说明外，所有内建函数在 GCC 支持的所有 C 语言族中均可使用。

    关键特性

    • 除具有库等价物的内建函数外，GCC 内建函数始终内联展开，没有对应的入口点，不能取其地址。
    • 在函数调用表达式之外使用内建函数会导致编译时错误。

    2. C 标准库对应的内建函数

    GCC 为大量 C 标准库函数提供了对应的内建版本，使编译器能够在适当时机将这些调用优化或内联展开。

    4. 栈分配内建函数

    __builtin_alloca

    在调用函数的栈帧上分配 size 字节的空间。与 VLA（变长数组）类似，分配的内存在函数返回时自动释放，不需要也不能调用 free() 。

    注意：__builtin_alloca 本质上等价于 C99 的 alloca，栈溢出风险较高，建议在明确知道分配大小的场景下谨慎使用。

    __builtin_alloca_with_align

    与 __builtin_alloca 类似，但允许指定对齐方式，align 以位（bit）为单位表示。

    __builtin_alloca_with_align_and_max

    在 __builtin_alloca_with_align 基础上增加了最大大小参数，供编译器进行静态分析。

    11. 对象大小检查

    GCC 支持编译时和运行时的对象大小检查，以防止缓冲区溢出。

    __builtin_object_size

    返回 ptr 指向的对象的大小（字节数）。type 参数控制计算方式：

    type 含义
    0 从 ptr 到对象末尾的最大字节数
    1 从 ptr 到子对象末尾的最大字节数
    2 从 ptr 到对象末尾的最小字节数
    3 从 ptr 到子对象末尾的最小字节数

    若大小不可确定，type 为 0 或 1 时返回 (size_t)-1，为 2 或 3 时返回 0。

    __builtin_dynamic_object_size

    与 __builtin_object_size 类似，但允许在运行时动态计算对象大小。

    13. 其他通用内建函数

    编译期信息查询

    __builtin_constant_p

    判断表达式 exp 在编译时是否为常量。返回 1 表示是编译期常量，0 表示不是（或无法确定）。常用于宏中根据参数是否为常量选择不同实现路径。

    __builtin_is_constant_evaluated（C++ only）

    在 constexpr 函数中判断当前是否处于常量求值上下文。这是 std::is_constant_evaluated() 的底层实现。

    __builtin_types_compatible_p（C only）

    判断两个类型是否兼容（忽略顶层 const/volatile 限定符）。返回 1 或 0。

    __builtin_choose_expr（C only）

    根据编译期常量 const_exp 的值选择 exp1 或 exp2，未选择的表达式不会被求值。与三元运算符 ?: 不同，未选中的分支不受类型提升约束。

    __builtin_has_attribute

    判断符号或类型是否声明了指定属性。

    分支预测提示

    __builtin_expect

    向编译器提供分支预测信息，提示 exp 的期望值为 c，默认概率为 90%。

    __builtin_expect_with_probability

    与 __builtin_expect 类似，但允许显式指定概率值（0.0 ~ 1.0）。

    程序控制

    __builtin_trap

    使程序异常退出。GCC 通过执行非法指令或调用 abort() 实现，具体机制因目标平台而异。常用于不可能到达的代码路径作为安全保障。

    __builtin_unreachable

    告知编译器此处代码永远不会被执行。有助于消除编译器警告、允许编译器做更积极的优化。

    对齐与内存提示

    __builtin_assume_aligned

    告知编译器指针至少按 align 字节对齐，允许编译器生成更优化的 SIMD 代码。可选第三个参数表示对齐偏移量。

    __builtin_prefetch

    向 CPU 发出预取指令，提前将数据加载到缓存中以减少访问延迟：

    • rw：0 表示预取用于读，1 表示预取用于写
    • locality：0~3，0 表示无时间局部性（用完即丢），3 表示极高时间局部性

    指令缓存刷新

    __builtin___clear_cache

    刷新 [begin, end) 内存范围内的处理器指令缓存。在动态代码生成（JIT）场景下必不可少——当程序向内存写入可执行代码后，必须调用此函数确保 CPU 能正确执行新写入的指令。

    类型与调试信息

    __builtin_LINE

    等价于 __LINE__ 预处理宏，返回调用处的行号。用作 C++ 函数默认参数时，返回调用方的行号。

    __builtin_FILE

    等价于 __FILE__ 预处理宏，返回源文件名。

    __builtin_FUNCTION

    等价于 __FUNCTION__，返回当前函数名。

    推测执行安全

    __builtin_speculation_safe_value

    用于缓解 Spectre 等推测执行漏洞（侧信道攻击）。在非推测执行时返回 val；在推测执行时，要么等待分支结果确认，要么在检测到错误推测时返回 failval。

    浮点数优化屏障

    __builtin_assoc_barrier

    阻止浮点表达式在结合律优化（-fassociative-math）和融合乘加（-ffp-contract=fast）下与相邻表达式合并。

    C++ 专用内建函数

    __builtin_bit_cast

    std::bit_cast<> 的底层实现，将 arg 的位模式重新解释为 type 类型。type 和 arg 的类型必须是可平凡复制（trivially copyable）且大小相同的类型。

    __builtin_tgmath（C/ObjC only）

    实现 <tgmath.h> 类型泛型宏的底层机制，根据参数类型自动选择合适的函数版本，避免宏参数的多次求值问题。

    内存与数据安全

    __builtin_clear_padding

    清除对象内部的填充位（padding bits），不影响值表示。在需要对对象进行按位比较（如原子操作）时非常有用，防止因未初始化的填充位导致比较失败。

    __builtin_counted_by_ref

    若指针指向的数组对象通过 counted_by 属性关联了元素计数对象，则返回该计数对象的指针；否则返回 NULL。

    14. 目标机器特定内建函数

    GCC 为各种目标架构提供了大量特定的内建函数，可以直接使用目标架构的特殊指令。

    x86 / x86-64

    x86 架构拥有最丰富的内建函数集，涵盖所有 SIMD 指令集扩展：

    指令集 寄存器/宽度 头文件 代表内建函数

    15. 最佳实践与注意事项

    何时使用内建函数

    1. 性能关键路径：在热点代码中使用位操作（__builtin_popcount、__builtin_clz）、预取（__builtin_prefetch）等。
    2. 多线程/并发代码：优先使用 __atomic_* 系列替代 __sync_* 系列，语义更清晰。
    3. 安全编码：使用溢出检查算术（__builtin_add_overflow 等）防止整数溢出漏洞。
    4. 分支预测：对已知概率分布的分支使用 __builtin_expect 改善分支预测。

    可移植性建议

    常见陷阱

    1. __builtin_clz(0) 和 __builtin_ctz(0) 的结果未定义，使用前必须检查参数不为零。
    2. __builtin_return_address(n)（n > 0） 在某些平台上可能崩溃，需要用 __builtin_frame_address(n) 先检查帧是否有效。
    3. 内建函数不能取地址，不能赋值给函数指针（有库等价物的除外）。
    4. 目标特定内建函数需要对应的编译选项（如 -mavx2、-msse4.2），否则会引发编译错误。
    5. __builtin_unreachable() 使用不当会导致未定义行为，只应放置在确实不可达的代码路径上。

    编译选项联动

    内建函数 相关编译选项
    SIMD 内建函数 -msse2, -mavx, -mavx2, -mavx512f
    __builtin_expect 效果 -fprofile-arcs 可覆盖
    __builtin_alloca -fstack-usage 可追踪栈用量
    _FORTIFY_SOURCE CHK 函数 -D_FORTIFY_SOURCE=2 -O1+
    FMA 指令 -mfma, -ffp-contract=fast
