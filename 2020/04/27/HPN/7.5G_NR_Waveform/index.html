    背景：什么是”传输波形”？

    在 5G 通信中，”传输波形”是指信号在空中接口（Uu 口）传输时所采用的调制与复用方式。它决定了：

    • 信号如何打包、调制后发送到空中；
    • 接收端如何从复杂的无线信道中准确还原数据；
    • 系统在多用户、多天线、多场景下的抗干扰能力与频谱效率。

    5G NR（New Radio）在设计上充分考虑了下行广播与手机接收的稳定性，以及上行终端的功耗约束，针对不同方向采用了不同但兼容的波形方案。

    一、下行传输波形：CP-OFDM

    下行（基站 → 手机） 统一采用 CP-OFDM（Cyclic Prefix Orthogonal Frequency Division Multiplexing，循环前缀正交频分复用）。

    1.1 OFDM：多子载波并行传输

    OFDM 的核心思想是将一个宽带信道分割为若干个窄带子载波，每个子载波独立传输一部分数据，并行发送。

    类比理解：把原本一条宽的”信号公路”，分成几十条、上百条窄车道，每一条车道（子载波）都单独传一部分数据，多车道公路自然比单车道公路走得快。

    特性 说明
    子载波数量 可达数百至数千条
    子载波间隔 15 kHz / 30 kHz / 60 kHz 等（由 μ 参数决定）
    正交性 各子载波频率点在彼此的零点上，频域互不干扰
    频谱效率 高，可紧密排列子载波，充分利用频谱资源

    OFDM 本身的局限在于：无线信道中存在多径传播（信号经墙壁、建筑物反射后走不同路径到达接收端，产生时延差），导致相邻符号之间的干扰（ISI）和子载波间的干扰（ICI），这正是 CP 出现的原因。

    1.2 循环前缀（CP）：信号的”缓冲垫”

    CP 的构造方式并非插入空白段，而是将每个 OFDM 符号的尾部复制一段，附加到符号头部，形成保护间隔。

    CP 的作用机制：

    1. 消除 ISI（符号间干扰）：只要最大多径时延差 ≤ CP 长度，前一个符号的多径残留只会落入后一个符号的 CP 区域，不会污染数据体。接收端丢弃 CP 部分，仅处理数据体，即可完整还原信号。

    2. 维持正交性，消除 ICI（载波间干扰）：CP 使每个符号看起来像”循环卷积”，在频域上保证各子载波之间的正交性不被破坏。这是选择”复制尾部”而非”空白填充”的根本原因。

    类比: 快递车（帮助理解多径干扰与 CP 的作用）

    把每一段信号比作一辆快递车，基站是仓库，手机是收件人，无线传播环境（建筑物、墙壁）是”复杂路况”：

    • 理想情况（只有直路）：快递车走一条直路匀速直达，手机看到”整辆车”到了，直接签收，信号完整无干扰。

    • 实际情况（多径传播）：除了直路，还有信号被墙壁反射走的”绕路”，绕路的快递车会晚到（反射信号比直射信号慢一步）。

      • 没有 CP：前一辆快递车的绕路车，会和后一辆快递车的直行车同时到达手机门口，两辆车挤在一起，手机分不清哪件快递是哪一车的 → 这就是符号间干扰（ISI）；有时候相邻车道的快递车也会互相蹭到 → 这就是载波间干扰（ICI）。

      • 有了 CP：相当于给每辆快递车的车头，贴了一段和车尾一样的”重复快递”（CP 段）。手机签收时，只认”缓冲垫后面的核心快递”（数据体），哪怕绕路的快递车晚到，撞到的也只是车头的缓冲垫，不会碰到底层的核心货物，手机就能准确收到信号。

    1.3 两种 CP 类型：适配不同传播环境

    5G NR 定义了两种 CP：

    CP 类型 长度 适用场景 适用 SCS
    Normal CP（正常 CP） 较短（约 4.7 μs @ 15 kHz） 城市、小区、写字楼等多径时延小的场景 所有 SCS（15/30/60/120/240 kHz）
    Extended CP（扩展 CP） 较长（约 16.67 μs） 郊区、农村、室内深覆盖（地下车库、楼道）等多径时延大的场景 仅 60 kHz SCS（μ=2）

    两种 CP 的通俗理解：

    • Normal CP（短缓冲垫）：适合城市、小区、写字楼这些”路况简单”的地方。信号传播路径少、延迟小，短缓冲垫足够用，既能防干扰，又不浪费频谱资源。日常刷手机、发消息基本都用这种。

    • Extended CP（长缓冲垫）：适合郊区、农村，或室内深度覆盖（比如地下车库、高楼楼道）这些”路况复杂”的地方。这里信号反射路径多、延迟差大，就像快递车要走很多弯路、晚到的时间更长，长缓冲垫才能更好地挡住干扰，避免信号”撞车”。

    说明：Extended CP 仅在 μ=2（60 kHz 子载波间隔）下支持，此时每符号持续时间较短，扩展 CP 才具备工程意义。同时，使用 Extended CP 时，上行须采用单端口（单天线）传输，长缓冲垫和多天线一起用会浪费频谱资源，得不偿失，需要在”防干扰”和”传输效率”之间找平衡。

    1.4 下行应用场景

    CP-OFDM 是所有下行场景的统一波形，包括：

    • 宏基站（Macro Cell）：覆盖城市主干道、大范围区域
    • 小基站（Small Cell）：补盲覆盖，如楼道、地下停车场
    • 室内分布系统（IDA/DAS）：商场、写字楼室内覆盖
    • 毫米波基站（FR2）：实验室、高端写字楼高速传输

    不论部署类型，手机侧接收统一使用 CP-OFDM，确保标准统一，设备兼容性好。

    二、上行传输波形：DFT-S-OFDM / CP-OFDM 可切换

    上行（手机 → 基站）采用可切换的波形设计，支持两种模式：

    • DFT-S-OFDM（DFT Spread OFDM，开启 DFT 扩展）：省电优先
    • CP-OFDM（关闭 DFT 扩展）：性能优先

    2.1 DFT 扩展的作用：降低 PAPR

    PAPR（Peak-to-Average Power Ratio，峰均功率比） 是描述信号波动幅度的指标：

    • PAPR 高：信号瞬时功率峰值远高于平均功率，功率放大器（PA）需要更大的线性范围，静态功耗大，效率低。
    • PAPR 低：信号功率平稳，PA 工作在高效率区间，终端耗电少。

    OFDM 本身 PAPR 较高（多个子载波叠加时可能出现高功率峰值）。引入 DFT 扩展后：

    1. 时域信号先经过 DFT（离散傅里叶变换），将数据”扩展”到频域；
    2. 扩展后的频域数据分配到各子载波，相当于把能量更均匀地分散；
    3. 经 IDFT（反离散傅里叶变换）合成时域信号后，PAPR 明显降低。

    类比：喊口号（帮助理解 PAPR 与 DFT 扩展）

    把手机发信号比作”喊口号”：

    • 高 PAPR（不开启 DFT 扩展）：相当于喊口号时”忽高忽低、落差极大”——突然大声嘶吼、突然小声低语。手机里的”发声器”（功率放大器）需要频繁调节音量，不仅设计复杂，还特别费电。

    • 低 PAPR（开启 DFT 扩展）：相当于”把口号的音量拉平”，让声音平稳、落差小。此时”发声器”不用频繁调节，设计更简单、耗电也更少——就像我们平稳说话比忽高忽低嘶吼更省力气一样。

    这种特性对普通手机、物联网设备（智能手表、智能门锁等）尤为关键，这些设备靠电池供电、功率有限，省电才是第一需求。

    2.2 两种上行模式对比

    特性 DFT-S-OFDM（开启 DFT 扩展） CP-OFDM（关闭 DFT 扩展）
    PAPR 低 高
    终端功耗 低（PA 效率高） 高（PA 需更大线性范围）
    覆盖距离 相对更远（相同功率下信噪比更高） 略短
    系统复杂度 发送端需额外 DFT 处理 与下行结构一致，实现简单
    多天线支持 受限（通常单层传输） 支持多层 MIMO 传输
    适用设备 手机、IoT 终端（电池供电、功率受限） 专业终端、工业设备（插电、不缺电）
    典型场景 普通用户上网、发消息、物联网 收银系统、工厂监控、企业专业终端

    2.3 模式切换机制：由基站通过 RRC 信令控制

    上行波形模式的切换不由终端自主决定，而是由基站通过 RRC（Radio Resource Control）信令下发指令控制：

    同一商场内的”按需分配”示例：

    设备 供电方式 基站指令 上行模式 效果
    顾客手机 电池供电，功率有限 开启 DFT 扩展 DFT-S-OFDM（省电） 刷手机、发消息不耗电太快
    商场收银终端 插电，不缺电 关闭 DFT 扩展 CP-OFDM（性能） 收银数据传输又快又稳，不卡顿
    工厂监控摄像头 插电，功率充足 关闭 DFT 扩展 CP-OFDM（性能） 视频流传输速率高，稳定不丢帧

    这种”按需分配”机制实现了不同设备类型的差异化服务，在兼顾覆盖与功耗的同时最大化频谱利用率。

    三、共享频谱下的 PRB 交织

    在多运营商共享频谱（如部分未授权频段或共享许可频段）场景下，多路信号共用同一段频谱，容易产生邻频干扰与同频碰撞。5G NR 在 FR1 频段（450 MHz–6 GHz） 的上行引入了 PRB 交织（PRB Interlacing） 机制来解决这一问题。

    3.1 什么是 PRB 交织？

    PRB（Physical Resource Block，物理资源块） 是 5G NR 中频域资源分配的基本单位，由连续的子载波组成。

    传统分配方式：连续占用一段 PRB，多用户/多运营商信号紧密相邻，容易产生干扰。

    PRB 交织的核心思路：

    将连续的 PRB 按固定间隔拆分为若干交织块（Interlace），每个交织块包含多个不连续的 PRB，分散在整个带宽中。

    停车场类比（帮助理解”分散占位”）

    原来停车场是连续的一排车位，不同的车紧挨着停，很容易互相蹭到（邻频干扰）。现在改成分散的多个车位——不同的车停在整个停车场的分散位置上，彼此之间隔着空位，就不容易互相碰到。同理，不同运营商的信号用分散的”交织块”传输，就不会出现邻频干扰和同频碰撞，信号传输更稳定。

    优势：

    • 分散式频率占用使相邻运营商/用户的信号在频域上不紧邻，降低邻频干扰；
    • 每个交织块覆盖全带宽，具备频率分集增益；
    • 便于多运营商在同一频段内灵活共享资源，互不影响。

    3.2 适用范围与限制

    条件 说明
    适用频段 仅 FR1（450 MHz–6 GHz），FR2（毫米波）不支持
    配置方式 由基站通过 SIB1（系统信息块1） 或专用 RRC 信令配置
    交织块参数 交织块数量、间隔大小由网络侧决定，终端按指示执行
    应用场景 NR-U（非授权频谱）、共享频谱场景，CBG（基于码块组）调度

    四、子载波间隔（SCS）与 CP 长度的关系

    5G NR 定义了一组数字学参数（Numerology），用参数 μ 描述子载波间隔（SCS）：

    μ SCS 典型应用 支持 Extended CP
    0 15 kHz 广覆盖、农村、FDD 否
    1 30 kHz 城市 Sub-6G 主力 否
    2 60 kHz 高速传输、室内 是
    3 120 kHz FR2 毫米波 否
    4 240 kHz FR2 参考信号 否

    SCS 与 CP 的关联规律：

    • SCS 越大（车道越宽），传输速度越快，符号时长越短（SCS=15kHz 时符号约 66.7 μs；SCS=120kHz 时约 8.3 μs）；
    • CP 长度随 SCS 增大而等比缩短（绝对时间），但相对符号时长的占比保持稳定（Normal CP 约 7%）；
    • 仅 SCS=60 kHz（μ=2）支持 Extended CP，此时 CP 长度约为符号时长的 25%，适应大时延扩展场景；
    • Extended CP 与上行多天线（多端口）传输不兼容，需在覆盖增强与传输效率之间权衡取舍。

    车道宽度类比：

    子载波就像公路车道，车道越宽（SCS 越大）传输越快，但需要的缓冲垫（CP）也相应调整。窄车道（15kHz）适合广覆盖（如农村大范围覆盖）；宽车道（60kHz 以上）适合高速传输（如城市里刷 4K 视频）。只有”60kHz 宽车道”（μ=2）才能用”长缓冲垫”（Extended CP），而且此时手机只能走单车道（单天线），因为长缓冲垫和多天线同时用会浪费公路资源，得不偿失。

    五、常见问题：为什么用复制尾部，而不是插入空白段？

    这是理解 CP 设计哲学的关键问题。

    空白段（Zero Padding Guard Interval）的缺陷

    若在符号间插入全零的空白段：

    1. 破坏循环卷积特性：OFDM 的频域正交性依赖于线性卷积被”循环化”的数学前提。空白段破坏了这一循环结构，接收端做 DFT 后各子载波不再正交。
    2. 产生 ICI：正交性一旦被破坏，子载波之间出现能量泄漏，形成载波间干扰（ICI），信号质量严重下降。

    类比：合唱团（空白段的问题）：

    把 OFDM 的各子载波比作合唱团的歌手，每个歌手唱自己的调，节奏完全对齐（正交），听众（手机）能清晰分辨每个声音——这是信号传输的理想状态。

    如果加空白段：相当于合唱团唱到一半，突然全体停唱几秒，再接着唱。停唱后再开口，歌手们的节奏全乱了，调也互相串，听众根本分不清谁唱谁——这就是”频域特性变差”，信号会失真、干扰严重。

    CP（复制尾部）的优势

    复制符号尾部贴到头部，等价于将线性卷积转化为循环卷积：

    1. 维持正交性：DFT 处理后子载波仍满足正交条件，频域特性完好；
    2. 消除 ISI：多径延迟落入 CP 区段，核心数据体不受污染；
    3. 实现简单：接收端仅需丢弃 CP 段，直接对数据体做 FFT，无需额外均衡处理。

    合唱团类比（CP 的优势）：

    如果加 CP：相当于合唱团每段开头，全体先重复一遍上一段的最后一句，再唱新内容。重复的部分不影响核心演唱，歌手们的节奏始终对齐，听众跳过重复的引子，依然能清晰分辨每个声音，不会破坏”秩序”（正交性）。

    代价与收益：

    空白段看似节省频谱资源，却会让信号混乱；CP 虽然多传了一段重复信号（约 7% 的时域开销），却能保证信号稳定，换来的信号质量提升远超这点开销，工程上完全值得。这就是工程师选择 CP 当缓冲垫的根本原因。

    总结

    维度 下行 上行
    波形 CP-OFDM（统一） DFT-S-OFDM 或 CP-OFDM（可切换）
    切换控制 无需切换 基站通过 RRC 信令控制
    核心优化目标 接收稳定性、频谱效率 终端功耗（PAPR）或传输性能
    CP 类型 Normal CP（通用）/ Extended CP（60kHz SCS） 同下行
    共享频谱优化 — PRB 交织（FR1 频段）

    5G NR 的波形设计体现了系统级的工程哲学：

    • 下行统一：降低手机实现复杂度，保障全场景兼容；
    • 上行灵活：兼顾功率受限的消费终端与性能优先的工业终端；
    • 资源管理精细化：通过 PRB 交织支持多运营商频谱共享，提升频谱利用效率。

    理解这些波形设计，有助于深入把握 5G NR 物理层架构的设计逻辑，也是进一步学习链路预算、MIMO、调度算法的重要基础。
