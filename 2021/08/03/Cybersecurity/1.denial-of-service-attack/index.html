    概述

    拒绝服务攻击（Denial-of-Service Attack，简称 DoS 攻击，读音 /dɒs/）是一种网络攻击手段，攻击者通过向目标机器或网络资源发送大量无效请求，试图使合法用户无法正常访问服务，从而临时或永久地中断连接在网络上的主机所提供的服务。

    DoS 攻击通常通过向目标机器或资源发送大量多余请求来实现，以期超载系统，阻止部分或全部合法请求得到响应。攻击范围相当广泛，从向服务器发送数百万个请求使其性能降低，到用大量无效数据压垮服务器，再到提交包含非法 IP 地址的请求，手段多种多样。

    形象比喻：DDoS 攻击类似于一群人拥堵在商店门口，使合法顾客难以进入，从而扰乱正常经营并造成商业损失。

    DoS 与 DDoS 的区别

    维度 DoS 攻击 DDoS 攻击
    来源 单一主机或少量来源 多个不同来源（通常 3-5 个节点以上）
    IP 地址 通常为单一或少量 IP 数千个感染恶意软件的主机 IP
    防御难度 相对简单，可屏蔽单一来源 复杂，仅屏蔽单一来源不够
    危害程度 较低 更高，难以追踪和停止
    典型工具 单机洪水工具 僵尸网络（Botnet）

    分布式拒绝服务攻击（Distributed Denial-of-Service，DDoS，读音 /ˈdiː.dɒs/）发生时，来自许多不同来源的入侵流量同时淹没受害目标。DDoS 攻击的规模在近年来持续上升，到 2016 年已超过每秒 1 Tbit（太比特）。

    攻击动机

    DoS/DDoS 攻击的动机多种多样：

    • 报复（Revenge）：针对特定组织或个人的报复行为
    • 勒索（Blackmail/Extortion）：攻击者威胁持续攻击，要求以比特币支付赎金
    • 黑客行动主义（Hacktivism）：以政治或社会目的为由发动的抗议攻击，例如匿名者（Anonymous）组织
    • 竞争性破坏：商业竞争对手雇用攻击者打击目标网站
    • 网络战（Cyber Warfare）：国家支持的行为者针对他国关键基础设施
    • 测试或炫耀技术：部分攻击出于技术炫耀或娱乐目的

    犯罪组织通常以银行、信用卡支付网关等高知名度 Web 服务器托管的网站或服务为目标。2015 年起，以 DD4BC 为代表的 DDoS 僵尸网络开始以金融机构为目标，惯用手法是先发动低烈度攻击，附带警告称若不支付比特币赎金将发动更大规模攻击。安全专家建议被攻击网站不要支付赎金，因为攻击者在意识到目标愿意付款后往往会变本加厉。

    历史事件与里程碑

    早期攻击

    • 1996 年 9 月 6 日：全球第三古老的互联网服务提供商 Panix 遭受 SYN 洪水攻击，服务中断数天，直到思科等硬件厂商研发出应对防御方案。此次事件中泄露的样本代码，随后被用于攻击 Sprint、EarthLink、E-Trade 等大型企业。
    • 2001 年初：微软（Microsoft）遭受 DoS 攻击，大量用户无法访问微软网站和相关服务。
    • 2014 年 6 月 10 日：Evernote 遭受长达 24 小时的 DDoS 攻击。

    攻击类型详解

    洪水攻击（Flood Attacks）

    最简单的 DoS 攻击依靠蛮力，向目标发送大量数据包，使其连接带宽饱和或耗尽系统资源。

    SYN 洪水攻击（SYN Flood）

    SYN 洪水发生时，主机发送大量 TCP/SYN 数据包，通常使用伪造的发送方地址。每个数据包都被当作连接请求处理，导致服务器产生半开连接（half-open connection），发回 TCP/SYN-ACK 数据包，并等待来自发送方地址的响应。然而由于发送方地址是伪造的，响应永远不会到来。这些半开连接耗尽服务器可用连接数，使其无法响应合法请求。

    SYN cookies 技术是应对 SYN 队列洪水的有效缓解手段，但无法解决带宽耗尽问题。

    2022 年，TCP 攻击是 DDoS 事件中使用最多的方式，占所有 DDoS 活动的 63% ，包括 TCP SYN、TCP ACK 和 TCP 洪水等战术。

    UDP 洪水攻击

    UDP 洪水是常见的 DDoS 攻击类型之一，通过向目标发送大量 UDP 数据包来消耗带宽。

    Ping 洪水（Ping Flood）

    向受害者发送压倒性数量的 ping 数据包，通常使用类 Unix 主机上的 ping 命令。主要要求是攻击者拥有比受害者更大的带宽。

    死亡 Ping（Ping of Death）

    向受害者发送格式错误的 ping 数据包，导致脆弱系统崩溃。

    应用层攻击（Application Layer Attacks）

    应用层 DDoS 攻击（也称第 7 层 DDoS 攻击）以应用层进程为目标，通过过度消耗网站的特定功能或特性使其失效。与整个网络攻击不同，它通常被用于对抗金融机构，以分散 IT 和安全人员对安全漏洞的注意力。

    • 2013 年：应用层 DDoS 攻击占所有 DDoS 攻击的 20%
    • 2013 年 Q4 至 2014 年 Q4：应用层攻击增加了 51%
    • 2014 年 Q3 至 Q4：再增加 16%
    • 2017 年：Cloudflare 工程师指出，尽管网络级攻击频率下降，但应用层攻击丝毫没有放缓迹象。

    HTTP 慢速 POST 攻击（Slow POST）

    首次发现于 2009 年。攻击者发送完整合法的 HTTP POST 头，其中包含 Content-Length 字段指定后续消息体大小，但随后以极低速率（如 1 字节/110 秒）发送实际消息体。服务器会遵从 Content-Length 字段等待完整消息体，攻击者建立数百甚至数千个此类连接，直到服务器上所有传入连接资源耗尽。

    该攻击的特点是针对逻辑资源而非网络带宽，因此受害者仍有足够带宽和 CPU，但连接池被耗尽。Apache HTTP Server 默认接受最大 2 GB 的请求，使此攻击极具威力。OWASP 发布了测试服务器对该类攻击防御能力的工具。

    CC 攻击（Challenge Collapsar）

    2004 年由中国黑客 KiKi 发明，针对 NSFOCUS 防火墙”Collapsar”而得名。CC 攻击向目标 Web 服务器频繁发送标准 HTTP 请求，这些请求的 URI 需要耗时的复杂算法或数据库操作，从而耗尽目标服务器资源。

    RUDY 攻击

    RUDY（R-U-Dead-Yet）针对 Web 应用，通过 Web 服务器上可用会话的耗尽来实现攻击。类似于 Slowloris，使用永无止境的 POST 传输来保持会话挂起，并发送任意大的 content-length 头值。

    Slowloris

    Slowloris 是一种慢速攻击工具，通过与受害者建立大量挂起连接来实现拒绝服务。其代码嵌入在恶意软件中，可在系统所有者不知情的情况下发动攻击。

    XDoS（XML DoS）

    一种可由现代 Web 应用防火墙（WAF）控制的应用级 DoS 攻击，属于超时利用类攻击。

    降级服务攻击（Degradation-of-Service）

    脉冲僵尸（Pulsing Zombies）是一组受控制的被入侵计算机，被指令对目标网站发动断断续续、短暂的洪水攻击，目的仅仅是降低其性能而非使其崩溃。这类攻击被称为”降级服务攻击”，比普通 DoS 更难检测，可长期持续地扰乱网站连接，造成的整体破坏可能更大。

    针对云端自动扩缩的 Yo-yo 攻击

    Yo-yo 攻击是一种针对使用自动扩缩（Autoscaling）的云托管应用程序的特定 DoS/DDoS 类型。攻击者在发送大量流量（导致向上扩展）和停止突发（导致向下缩减）之间反复切换，通过频繁触发扩缩操作向应用运营商增加额外云计算费用，或使其陷入持续的资源调度混乱。

    协议漏洞攻击

    Smurf 攻击

    Smurf 攻击依赖于错误配置的网络设备，允许数据包通过网络广播地址发送到特定网络上的所有计算机。攻击者发送大量 IP 数据包，其源地址被伪造成受害者地址，网络上的大多数设备会向该源 IP 地址回复，导致受害者计算机被流量淹没，甚至无法使用。

    Ping 洪水是 Smurf 攻击的基础形式之一。

    Fraggle 攻击

    类似于 Smurf 攻击的带宽消耗攻击。

    泪滴攻击（Teardrop Attack）

    向目标机器发送带有重叠、超大载荷的破损 IP 分片。这会导致各种操作系统崩溃，因为其 TCP/IP 分片重组代码存在漏洞。Windows 3.1x、Windows 95、Windows NT 以及 2.0.32 和 2.1.63 版本之前的 Linux 均易受此攻击。

    Nuke（核弹攻击）

    一种老式拒绝服务攻击，通过向目标发送分片的或无效的 ICMP 数据包，使用改装的 ping 工具反复发送这些损坏数据，使受影响的计算机速度减慢直到完全停止。典型案例是 WinNuke，利用了 Windows 95 中 NetBIOS 处理程序的漏洞，向受害者机器的 TCP 139 端口发送带外数据，导致系统锁死并显示蓝屏死机（BSOD）。

    BlackNurse 攻击

    利用目标端口不可达 ICMP 数据包的一种攻击。

    剪切攻击（Shrew Attack）

    针对传输控制协议（TCP）的拒绝服务攻击，攻击者采用中间人技术，利用 TCP 重传超时机制的弱点，使用短暂的同步突发流量破坏同一链路上的 TCP 连接。

    SACK Panic（Linux 内核漏洞）

    通过操纵最大分段大小（MSS）和选择性确认（SACK）触发 Linux 内核的整数溢出，可能导致内核崩溃（kernel panic）。2019 年 6 月 17 日，Jonathan Looney 发现了相关 CVE（CVE-2019-11477、CVE-2019-11478、CVE-2019-11479）。

    UPnP 攻击

    利用通用即插即用（UPnP）协议中的现有漏洞，绕过网络安全并向目标网络和服务器发动洪水攻击，基于 DNS 放大技术，但攻击机制是将请求从一个外部来源转发到另一个来源的 UPnP 路由器。返回的数据来自意外 UDP 端口和虚假 IP 地址，使流量难以被简单封堵。

    SSDP 反射攻击

    2014 年发现简单服务发现协议（SSDP）被用于 DDoS 攻击。许多设备（包括家用路由器）的 UPnP 软件中存在漏洞，允许攻击者将来自 UDP 端口 1900 的回复发送到其选择的目的地址。借助数千台设备的僵尸网络，攻击者可以产生足够的数据包速率和占用带宽来饱和链路。Cloudflare 将 SSDP 描述为”愚蠢简单的 DDoS 协议”（Stupidly Simple DDoS Protocol）。

    ARP 欺骗（ARP Spoofing）

    常见的 DoS 攻击，利用 ARP 协议中的漏洞，使攻击者能将其 MAC 地址与另一台计算机或网关的 IP 地址关联，导致发往原始 IP 的流量被重定向到攻击者，从而造成拒绝服务。

    TTL 过期攻击

    丢弃 TTL 值为 1 或更低的数据包比转发 TTL 较高的数据包需要更多路由器资源。当数据包因 TTL 到期被丢弃时，路由器 CPU 必须生成并发送 ICMP 超时响应。大量产生这些响应会使路由器 CPU 过载。

    放大/反射攻击（Amplification & Reflection）

    分布式反射拒绝服务（DRDoS）攻击：攻击者向大量计算机发送伪造请求，使用目标受害者的 IP 地址作为源地址，所有回复将涌入并淹没目标。

    放大攻击用于放大发送给受害者的带宽。US-CERT 观察到不同服务可能产生不同的放大系数：

    DNS 放大攻击

    攻击者向一个或多个公共 DNS 服务器发送 DNS 名称查询请求，将目标受害者的源 IP 地址欺骗为发送方。攻击者尽量请求尽可能多的信息，从而放大发送给目标受害者的 DNS 响应，大幅增加针对目标的流量。

    NTP 放大攻击

    利用 NTP 服务器的 monlist 命令（返回请求过时间的最后 600 个主机的详细信息）。向该时间服务器发送伪造源 IP 地址的小请求，将收到 556.9 倍大小的响应发送给受害者。与僵尸网络结合后，所有发送相同欺骗 IP 源请求的节点将导致大量数据被发回受害者，且响应数据来自合法服务器，极难防御。

    永久性拒绝服务攻击（PDoS）

    永久性拒绝服务攻击（Permanent Denial-of-Service，PDoS），也称为”phlashing“，是一种损坏系统严重到需要更换或重新安装硬件的攻击。

    PDoS 攻击利用受害者硬件（路由器、打印机或其他网络硬件）管理接口的远程管理安全漏洞，将设备固件替换为修改过的、损坏的或有缺陷的固件镜像——即”刷机”操作。目的是将设备变砖（brick），使其无法用于原始用途，直到修复或更换。

    PDoS 是纯粹针对硬件的攻击，比 DDoS 攻击更快速，所需资源更少。

    • BrickerBot：一款针对 IoT 设备的恶意软件，使用 PDoS 攻击禁用目标设备。
    • PhlashDance：由 HP 系统安全实验室员工 Rich Smith 创建的工具，用于在 2008 年伦敦 EUSecWest 应用安全会议上检测和演示 PDoS 漏洞。

    高级持续性拒绝服务攻击（APDoS）

    高级持续性拒绝服务攻击（Advanced Persistent DoS，APDoS）与高级持续性威胁（APT）相关联，需要专门的 DDoS 缓解措施。

    记录在案的最长连续攻击持续了 38 天，涉及约 50+ Pb（50,000+ TB）的恶意流量。攻击者可能战术性地在多个目标之间切换，制造干扰以规避 DDoS 防御措施，但最终将主要攻击力量集中在单一受害者身上。

    APDoS 攻击的特征：

    • 高级侦察：攻击前进行开源情报（OSINT）收集和大量诱饵扫描，以长期规避检测
    • 战术执行：同时攻击主要和次要目标，但焦点在于主要目标
    • 明确动机：有精心设计的最终目标
    • 强大计算能力：具备访问大量计算机能力和网络带宽
    • 多线程 OSI 层攻击：在 OSI 第 3 层至第 7 层同时运作的复杂工具
    • 长期持续性：将上述所有特点整合成跨多目标的协调、精心管理的攻击

    其他特殊攻击类型

    点对点 DDoS（Peer-to-Peer DDoS）

    攻击者利用对等网络服务器中的漏洞发动 DDoS 攻击。最具攻击性的此类攻击利用 DC++ 协议。攻击者充当”幕后操控者”，指示大型点对点文件共享中心的客户端断开与对等网络的连接，转而连接受害者的网站，无需僵尸网络，也无需与被颠覆的客户端通信。

    香蕉攻击（Banana Attack）

    一种特殊类型的 DoS 攻击，通过将客户端发出的消息重定向回客户端本身，既阻止外部访问，又用发送的数据包淹没客户端。LAND 攻击属于此类型。

    马尔可夫调制拒绝服务攻击（Markov-Modulated DoS）

    攻击者使用隐马尔可夫模型破坏控制数据包。在在线游戏等场景中较为常见，控制数据包的破坏会破坏游戏进程和系统功能。

    电话拒绝服务攻击（TDoS）

    语音网络电话（VoIP）使大量语音呼叫的发起成本低廉且易于自动化，同时允许通过来电显示欺骗（Caller ID Spoofing）伪造呼叫来源。FBI 记录的相关欺诈场景包括：

    • 诈骗者联系受害者的银行或经纪商，伪装成受害者请求转账，同时用虚假来电淹没受害者电话线，使其无法被联系到
    • 诈骗者向消费者谎称追收发薪日贷款，当消费者反对时，用自动呼叫淹没受害者雇主
    • 骚扰性攻击（Swatting）：诈骗者以收债为由威胁报警，当受害者拒绝时，用来电显示欺骗成受害者号码淹没当地警局，警察随后上门

    历史案例：2002 年新罕布什尔州参议院选举电话堵塞丑闻中，电话推销员被用于向政治对手的电话热线发送大量虚假来电。

    Booter / Stresser 服务

    部分供应商提供所谓的”booter”或”stresser”服务，这些服务具有简单的基于 Web 的前端，并通过网络接受付款。以压力测试工具为名进行营销，实际上可用于执行未经授权的拒绝服务攻击，允许技术能力有限的攻击者访问复杂的攻击工具。由僵尸网络驱动，此类工具产生的流量通常在 5–50 Gbit/s 之间，足以使普通家庭用户无法上网。

    僵尸网络与攻击基础设施

    僵尸网络（Botnet） 是被入侵计算机的集合，是现代 DDoS 攻击的核心基础设施。

    典型工具

    • Stacheldraht：经典 DDoS 工具，使用分层结构——攻击者通过客户端程序连接到处理程序（被入侵系统），处理程序向僵尸代理发出命令，每个处理程序可控制多达 1000 个代理
    • Low Orbit Ion Cannon（LOIC） 和 High Orbit Ion Cannon（HOIC）：常用于黑客行动主义攻击（如匿名者的”Operation Payback”行动），部分情况下机器所有者知情并同意参与
    • MyDoom：DoS 机制在特定日期和时间触发，目标 IP 地址在发布前硬编码，无需进一步交互即可发动攻击

    Mirai 僵尸网络

    Mirai 通过计算机蠕虫感染数十万台 IoT 设备（温控器、Wi-Fi 时钟、洗衣机等保护不足的设备），设备所有者通常无法立即察觉设备被感染。2016 年 10 月，Mirai 僵尸网络攻击了 Dyn（Netflix 等网站的 DNS 提供商），导致这些网站数小时不可访问。

    IoT 设备引发的新型攻击也在增加，曾记录到约 900 台 CCTV 摄像头发起的峰值约 2 万请求/秒的攻击。

    英国 GCHQ 也拥有名为 PREDATORS FACE 和 ROLLING THUNDER 的 DDoS 工具。

    非蓄意拒绝服务

    非蓄意拒绝服务（Unintentional DoS）不是由蓄意攻击造成的，而是由于突然的巨大流量激增导致系统被拒绝访问。

    常见场景：

    • 一个极受欢迎的网站在新闻报道中发布了指向一个准备不足的次要网站的显著链接，导致该次要网站在几个小时内被数十万用户点击，效果等同于 DDoS 攻击，也称为”Slashdot 效应“
    • VIPDoS：当链接由名人发布时，由此产生的流量激增尤为剧烈
    • 搜索引擎抓取：搜索引擎频繁地对尚未准备好处理高流量的小站点进行索引

    典型案例：2009 年迈克尔·杰克逊逝世时，谷歌、Twitter 等网站显著减速甚至崩溃，许多网站服务器误认为这些请求来自病毒或间谍软件。

    攻击症状与检测

    美国计算机应急准备小组（US-CERT）识别出的拒绝服务攻击症状包括：

    • 网络性能异常缓慢（打开文件或访问网站）
    • 特定网站无法访问
    • 无法访问任何网站

    对于云端应用，基于应用层分析的检测方法可判断大量入站流量是否合法。这些方法主要依赖于在应用程序内识别出的价值路径，通过”关键完成指标”（Key Completion Indicators）标记来监控请求进度。本质上是通过统计方法评估入站请求行为，检测是否存在异常。

    防御手段

    防御 DoS/DDoS 攻击通常需要综合使用攻击检测、流量分类和响应工具，旨在拦截被识别为非法的流量，同时允许合法流量通过。

    1. 流量清洗中心（Scrubbing Center）

    所有发往受害者的流量被转移，通过各种方式（DNS 系统中更改受害者 IP 地址、隧道方法 GRE/VRF、MPLS、SDN、代理、数字交叉连接或直接电路）经过清洗中心。清洗中心分离恶意流量，仅将合法流量转发给受害者服务器。

    2. 应用前端硬件（Application Front-End Hardware）

    放置在流量到达服务器之前的网络上的智能硬件，可与路由器和交换机配合使用，分析进入网络的数据包，识别并丢弃危险或可疑流量。

    3. 黑洞路由（Blackhole Routing）

    将对攻击 DNS 或 IP 地址的所有流量发送到黑洞（空接口或不存在的服务器）。可由 ISP 管理以提高效率并避免影响网络连接。

    4. DNS 沉洞（DNS Sinkhole）

    将流量路由到一个分析流量并拒绝恶意数据包的有效 IP 地址。对严重攻击可能效率不足。

    5. 入侵防御系统（IPS）

    IPS 在攻击具有关联签名时有效。但基于内容识别的 IPS 无法拦截基于行为的 DoS 攻击。基于 ASIC 的 IPS 具备检测和拦截 DoS 攻击的处理能力，可自动作为断路器运作。

    6. DoS 防御系统（DDS）

    比 IPS 更专注于该问题，DDS 可拦截基于连接的 DoS 攻击和内容合法但意图恶意的攻击，同时可应对协议攻击（如泪滴攻击、死亡 Ping）和基于速率的攻击（如 ICMP 洪水、SYN 洪水）。目的构建的系统可以比基于软件的系统更快识别和阻止拒绝服务攻击。

    7. 防火墙（Firewall）

    在简单攻击的情况下，可调整防火墙基于协议、端口或来源 IP 地址拒绝所有来自攻击者的入站流量。然而对于复杂攻击则较难，防火墙也可能位于网络层次结构的深处，路由器在流量到达防火墙之前就受到不利影响。许多安全工具仍不支持 IPv6，可能被攻击绕过。

    8. 路由器（Router）

    路由器具有速率限制和 ACL 能力，但需手动设置，且在 DoS 攻击下容易被压垮。Nokia SR-OS 使用 FP4 或 FP5 处理器提供 DDoS 保护，同时使用基于大数据分析的 Nokia Deepfield Defender。Cisco IOS 有可选功能可减少洪水影响。

    9. 交换机（Switch）

    大多数交换机具有速率限制和 ACL 能力。部分高端交换机提供自动或全系统速率限制、流量整形、延迟绑定（TCP 拼接）、深包检测（DPI）和 bogon 过滤，以自动检测和修复 DoS 攻击：

    • SYN 洪水：可用延迟绑定或 TCP 拼接防止
    • 基于内容的 DoS：可用深包检测防止
    • 火星包攻击：可用 bogon 过滤防止

    10. 端口封锁

    威胁可能与特定 TCP 或 UDP 端口号相关联，在防火墙封锁这些端口可缓解攻击。例如在 SSDP 反射攻击中，关键缓解措施是封锁端口 1900 上的入站 UDP 流量。

    11. TTL 过期封锁

    根据网络路径长度封锁特定 TTL 值，可作为封锁欺骗攻击的可行选项。

    法律与监管

    在许多国家，发动 DoS 攻击属于违法行为。

    国家/地区 相关法律
    美国 《计算机欺诈和滥用法》（CFAA）第 18 章 1030 节，可处以最高 10 年监禁
    欧盟 《网络攻击指令》（2013/40/EU），要求成员国将 DDoS 攻击列为刑事犯罪，处以至少 2 年监禁
    英国 《计算机滥用法》1990，可处以最高 10 年监禁
    德国 《刑法典》第 303b 条，可处以最高 3 年监禁，情节严重者最高 10 年
    中国 《网络安全法》及相关刑法条款，可处以拘留或有期徒刑

    值得关注的是，部分 DoS 攻击工具（如 LOIC）被黑客行动主义者使用，在一些判例中法庭需要判断参与者是否自愿参与以及是否构成实质损害。
